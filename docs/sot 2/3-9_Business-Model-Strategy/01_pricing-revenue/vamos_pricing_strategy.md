## S7H-011. Pay-as-you-go 핵심 전략 — 사용한 만큼만 ★CRITICAL

> **우선도**: CRITICAL | **버전**: V1

VAMOS V1의 핵심 가치 제안: **"구독이 아닌, 사용한 만큼만"**

> **LOCK-BM-04 준수**: 과금 단위 = 요청별 토큰 기반 (API) + 월 구독 (SaaS)

```

  ChatGPT Plus: $20/mo 고정 (안 써도 $20)
  Claude Pro:   $20/mo 고정 (안 써도 $20)
  Cursor Pro:   $20/mo 고정 (안 써도 $20)

VAMOS 모델:
  V1 Core:     API 사용량 × 단가 (안 쓰면 $0)

## S7H-012. 비용 투명성 — 경쟁 차별화 핵심 ★CRITICAL

> **우선도**: CRITICAL | **버전**: V1

### 12.1 비용 투명성 정의

모든 LLM API 요청의 비용을 실시간으로 투명하게 공개하는 시스템. 어떤 경쟁 AI 서비스도 이 수준의 비용 투명성을 제공하지 않음 (STEP7-H S7H-012 차별화 명시).

### 12.2 비용 투명성 구현 체계

| 시점 | 기능 | 사용자 경험 |
|------|------|-----------|
| **요청 전** | 예상 비용 표시 | "이 작업의 예상 비용: ₩15~25" |
| **요청 중** | 실시간 토큰 카운트 + 비용 | 실시간 진행 표시 |
| **요청 후** | 실제 비용 확정 | "이 응답 비용: ₩18 (입력 500tok + 출력 1,200tok)" |
| **월간 리포트** | 절감 리포트 | "스마트 라우팅으로 이번 달 ₩12,000 절감" |

> **출처**: STEP7-H S7H-012 구현 항목 전수 반영

### 12.3 비용 표시 UI 스키마

```yaml
cost_display:
  pre_request:
    estimated_cost_range:
      min: number  # KRW
      max: number  # KRW
    model_options:
      - model: string
        cost: number
        quality: string  # "high" | "medium" | "low"
  
  post_request:
    actual_cost:
      input_tokens: number
      output_tokens: number
      model_used: string
      cost_krw: number
      cost_usd: number
    savings:
      vs_direct_api: number  # 직접 호출 대비 절감
      vs_subscription: number  # $20/mo 구독 대비 절감
  
  monthly_report:
    total_cost: number
    total_requests: number
    avg_cost_per_request: number
    model_distribution: map<string, number>
    savings_from_routing: number
    savings_vs_subscription: number
```

### 12.4 경쟁사 비용 투명성 비교

| 서비스 | 비용 사전 표시 | 비용 사후 표시 | 절감 리포트 | 모델 비용 비교 |
|--------|-------------|-------------|-----------|-------------|
| ChatGPT Plus | ❌ | ❌ | ❌ | ❌ |
| Claude Pro | ❌ | ❌ | ❌ | ❌ |
| Cursor | ❌ | ❌ | ❌ | ❌ |
| **VAMOS** | **✅** | **✅** | **✅** | **✅** |

> **차별화**: VAMOS만이 유일하게 4단계 비용 투명성 제공 (STEP7-H S7H-012)

## S7H-013. 비용 제어 — 사용자가 완전히 통제 ★CRITICAL

> **우선도**: CRITICAL | **버전**: V1

### 13.1 비용 제어 시스템

사용자가 AI 비용을 완전히 통제할 수 있는 시스템. 핵심 메시지: **"절대 예상치 못한 AI 비용 청구 없음"** (STEP7-H S7H-013 명시).

> **LOCK-BM-01 준수**: V1 월 비용 상한 ₩40,000 (~$30) — 시스템 하드캡

### 13.2 비용 제어 기능 상세

| 기능 | 설명 | 기본 값 |
|------|------|--------|
| 일일 예산 하드캡 | 일일 최대 지출 한도 설정 | ₩5,000/일 (사용자 조절 가능) |
| 주간 예산 | 주간 지출 한도 | ₩20,000/주 (사용자 조절 가능) |
| 월간 예산 하드캡 | 월간 최대 지출 한도 (★상위 우선) | ₩40,000/mo (LOCK-BM-01 절대 상한) — 일/주 하위 캡보다 우선. 월 누적 ₩40,000 도달 시 일/주 잔여와 무관하게 유료 API 차단(월 누적 체크가 일·주 캡을 override) |
| 알림 임계값 | 예산 소진 알림 | 50%, 80%, 95% 도달 시 알림 |
| 자동 라우팅 전환 | 예산 잔여 적을 때 저비용 모델 자동 전환 | 90% 소진 시 자동 전환 |
| 사용 패턴 분석 | 비용 절감 추천 | 주 1회 분석 리포트 |

### 13.3 비용 제어 UI 흐름

```
사용자 설정:
  ┌─────────────────────────────────┐
  │ 월간 예산: ₩40,000 (최대)       │
  │ 일일 예산: ₩5,000              │
  │ 알림:  50% ☑  80% ☑  95% ☑    │
  │ 자동 절약 모드: ON             │
  └─────────────────────────────────┘

실시간 표시:
  ┌─────────────────────────────────┐
  │ 이번 달: ₩12,340 / ₩40,000     │
  │ ████████░░░░░░░░░░  31%        │
  │ 오늘:    ₩1,230 / ₩5,000       │
  │ ███░░░░░░░░░░░░░░░  25%        │
  └─────────────────────────────────┘
```

### 13.4 예산 초과 방지 정책

| 상황 | 시스템 동작 | 사용자 선택 |
|------|-----------|-----------|
| 50% 소진 | 알림 발송 | 계속 / 절약 모드 전환 |
| 80% 소진 | 경고 + 절약 모드 권유 | 계속 / 절약 모드 / 일시 정지 |
| 95% 소진 | 강력 경고 + 자동 절약 모드 | 계속 (확인 필요) / 정지 |
| 100% 소진 (하드캡) | 유료 API 차단, 로컬 LLM만 가용 | 추가 예산 설정 / 다음 달 대기 |

> **LOCK-BM-01**: V1 월 비용 상한 ₩40,000 — 시스템 하드캡으로 이 금액 초과 불가

### 14.4 로컬 실행의 기술적 장점

- Knowledge Graph 데이터가 로컬에만 존재 → 관계 데이터 유출 불가
- 5-Layer 메모리 전체가 로컬 → 개인 컨텍스트 완전 보호
- API 호출 시에도 로컬 전처리 → 민감 정보 마스킹 가능

#### §E.1.4 전환 퍼널별 정량 지표 (Y1~Y2 기준)

| 전환 구간 | Y1 (5,000 사용자) | Y2 (30,000 사용자) | 산출 근거 |

> **가정 (E7)**: Y1 Pro 105명 × $15 × 12 = **$18,900** ARR (Pro 한정). API 마진 + Power + Team 포함 시 S7H-037 기준 시나리오 Y1 $600K 목표의 일부.
