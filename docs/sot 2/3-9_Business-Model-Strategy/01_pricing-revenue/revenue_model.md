## S7H-019. API 마진 모델 — V1 수익화 시작점 ★CRITICAL

> **우선도**: CRITICAL | **버전**: V1

### 19.1 V1 API 마진 모델 개요

VAMOS를 통한 LLM API 호출에 소폭 마진을 부과하여 V1 초기 수익화를 시작한다.

> **LOCK-BM-04 준수**: 과금 단위 = 요청별 토큰 기반 (API)
> **LOCK-BM-09 준수**: 마켓플레이스 수수료 = 70% 개발자 / 30% VAMOS

### 19.2 API 마진 구조 (STEP7-H S7H-019 원본)

| 모드 | 설명 | 마진율 | V1 기본 |
|------|------|--------|--------|
| **직접 API** | 사용자가 자기 API 키 입력 | $0 (마진 없음) | ✅ 기본 모드 |
| **VAMOS Proxy** | VAMOS API 키 사용 → 마진 부과 | 10~20% | ✅ 선택 모드 |

> **출처**: STEP7-H S7H-019 "직접 API: 사용자가 자기 API 키 입력 → 마진 $0 (V1 기본) / VAMOS Proxy: VAMOS API 키 사용 → 10-20% 마진"

### 19.4 사용자 가치 제안 (Proxy 모드)

사용자가 VAMOS Proxy를 선택하는 이유:

| 가치 | 설명 |
|------|------|
| 단일 API 키 | 여러 프로바이더를 하나의 VAMOS 키로 사용 |
| 자동 라우팅 | 질문 복잡도에 따라 최적 모델 자동 선택 |
| 비용 최적화 | 스마트 라우팅으로 불필요한 고비용 호출 방지 |
| 간편 결제 | 하나의 결제 수단으로 모든 프로바이더 통합 |
| 비용 투명성 | 요청별 실시간 비용 표시 (S7H-012 연계) |
