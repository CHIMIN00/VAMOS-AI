## S7H-065: API 가격 리스크 — 가격 변동 대응 [CRITICAL, V1]

### 리스크 정의

LLM API 프로바이더(OpenAI, Anthropic, Google 등)의 가격 인상 또는 무료 Tier 폐지로 인해 VAMOS 서비스의 운영 비용이 급등하고, 사용자 비용 상한(LOCK-BM-01~03)을 초과하는 시나리오.

> **출처**: STEP7-H S7H-065 — "LLM API 가격 인상 또는 무료 Tier 폐지 리스크"

### 영향 분석

| 항목 | 값 |
|------|---|
| **발생 확률** | MED — 2024-25 추세는 가격 하락이나, 프로바이더 정책 변경 가능성 존재 (출처: STEP7-H S7H-065) |
| **영향도** | VERY HIGH — LOCK-BM-01 V1 월 비용 상한 ₩40,000(~$30), LOCK-BM-02 V2 ₩93,000(~$70), LOCK-BM-03 V3 ₩266,000(~$200) 초과 시 가격 정책 재설계 불가피 |
| **종합 등급** | **CRITICAL** |

### 시나리오 분석

#### 시나리오 1: API 가격 50% 인상

- **현재 V1 월 비용**: ~$34/mo (S7H-073 기준, API 테스트 ~$20 포함)
- **인상 후**: API 비용 $20 → $30, 총 $44/mo
- **LOCK-BM-01 위반 여부**: $44는 **운영 비용**(API 원가)이며 LOCK-BM-01 ₩40,000은 **사용자 과금 상한**(사용자 비용 보호) — 별개 개념(financial_projection.md §1 L49 정합). 운영 비용 상승이 최종 사용자 과금을 ₩40,000 초과로 강제할 때만 위반. 본 시나리오는 **운영 마진 리스크**(LOCK 위반 아님)
- **마진 영향**: 사용자당 API 마진 $0.80 → $0.40 (50% 감소)

#### 시나리오 2: 무료 Tier 폐지

- **현재 전략**: Free Tier에서 로컬 LLM + 무료 API 50회/일 (LOCK-BM-06)
- **폐지 시**: 무료 API 대안 필요 → 로컬 LLM 전용 모드로 전환
- **사용자 경험 영향**: 응답 품질 저하 가능, Free→Core 전환율 감소 우려

#### 시나리오 3: 프로바이더 독점 강화

- **현재**: 멀티 프로바이더 전략으로 분산
- **독점 시**: 특정 프로바이더 의존도 증가 → 협상력 약화

### 대응 전략

| 전략 | 설명 | 실행 시점 |
|------|------|----------|
| **멀티 프로바이더 전략** | 단일 API 의존 제거, 최소 3개 프로바이더 병렬 지원 | V1~ (즉시) |
| **로컬 LLM Fallback** | 항상 무료 대안(Ollama, llama.cpp 등) 유지, 오프라인 모드 보장 | V1~ (즉시) |
| **가격 모니터링 시스템** | API 가격 변동 감지 시 자동 라우팅 조정 (최저가 프로바이더 우선) | V1~ (즉시) |
| **장기 계약·볼륨 할인** | 사용량 증가 시 프로바이더별 볼륨 할인 협상 | V2+ |
| **자체 Fine-tune 모델** | 핵심 기능용 자체 모델로 API 의존도 점진 감소 | V2+ |

> **완화 요인**: 2024-25 추세는 가격 하락 → 유리한 환경 (출처: STEP7-H S7H-065)

### 잔여 리스크

- 로컬 LLM 품질이 상용 API 대비 낮을 수 있음 → 사용자 만족도 하락 가능
- 멀티 프로바이더 전략의 통합 복잡성 증가
- **LOCK-BM-01~03 비용 상한은 절대 불변** — 상한 초과 시 기능 축소로 대응 (가격 인상 불가)

## S7H-066: 경쟁 리스크 — Big Tech의 무료화 전략 [CRITICAL, V1]

### 리스크 정의

Google, Meta, Microsoft 등 대형 플레이어가 AI 비서 서비스를 무료로 제공하거나, 오픈소스 프로젝트가 VAMOS의 핵심 기능을 대체하는 시나리오.

> **출처**: STEP7-H S7H-066 — "Google/Meta 등이 무료 AI 서비스 확대 시"

### 영향 분석

| 항목 | 값 |
|------|---|
| **발생 확률** | HIGH — Google Gemini, Meta Llama 등 이미 무료/저가 모델 공격적 확대 중 (2025년 기준) |
| **영향도** | VERY HIGH — Free/Core Tier 사용자 이탈, 유료 전환율(7% 목표) 하락 가능 |
| **종합 등급** | **CRITICAL** |

### 시나리오 분석

#### 시나리오 1: Big Tech 무료 AI 비서 출시

- Google/Apple이 OS 내장 AI 비서를 무료 제공
- VAMOS Free Tier의 차별성 약화
- **예상 영향**: 신규 사용자 유입 30~50% 감소 (추정, 유사 시장 사례 기반)

#### 시나리오 2: 오픈소스 경쟁 프로젝트 부상

- VAMOS와 유사한 오픈소스 개인 AI 비서 프로젝트 등장
- 커뮤니티 기여자 분산 가능
- **예상 영향**: 기술적 우위 기간 6~12개월로 단축

#### 시나리오 3: 가격 전쟁 (Pro Tier 경쟁)

- 경쟁 서비스가 $15/mo(LOCK-BM-07) 이하 가격으로 유사 기능 제공
- VAMOS Pro Tier 가격 경쟁력 약화
- **예상 영향**: Pro 전환율 7% → 3~4%로 하락 가능 (추정)

### 대응 전략

| 전략 | 설명 | 실행 시점 |
|------|------|----------|
| **데이터 주권 포지셔닝** | "무료 서비스의 대가는 당신의 데이터" 메시지 — Big Tech 대비 프라이버시 우위 | V1~ (즉시) |
| **개인화 차별화** | 로컬 데이터 기반 개인화는 Big Tech이 구조적으로 제공 불가한 가치 | V1~ (즉시) |
| **오픈소스 + 커뮤니티** | 오픈소스로 커뮤니티 충성도 구축, 기여자 생태계로 방어 | V1~ (즉시) |
| **니치 시장 집중** | 투자 분석, 프라이버시 중시 사용자 등 니치 세그먼트 집중 공략 | V1~ (즉시) |
| **전환 비용 구축** | 사용 기간 증가에 따른 개인화 데이터 축적 → 자연스러운 Lock-in | V2+ |

> **출처**: STEP7-H S7H-066 대응 전략 전문

### 잔여 리스크

- Big Tech의 자본력 대비 마케팅·브랜드 인지도 절대 열위
- 니치 시장 규모가 BEP 달성에 불충분할 가능성
- 오픈소스 전략은 수익화와 상충할 수 있음 → 7티어 구조(C-7 해결)로 Free/유료 경계 관리
