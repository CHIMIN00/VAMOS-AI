## S7H-027 | CRITICAL | V1 | P1: 테크 얼리어답터 — 핵심 초기 사용자

> ⚠️ CONFLICT RESOLVED (C-5): STEP7-H 기준 "테크 얼리어답터" 적용. 상세명세의 "인디 개발자" 분류는 하위 참조.

### 인구통계

| 항목 | 상세 |
|------|------|
| 연령대 | 25~40세 |
| 직종 | 소프트웨어 엔지니어, 풀스택 개발자, ML/AI 엔지니어, DevOps |
| 기술 수준 | 상급 — CLI/터미널 환경에 익숙, 오픈소스 프로젝트 기여 경험 |
| 소득 수준 | 중~중상 (연봉 5,000~1억 원 또는 $50K~$100K) |
| 지리 | 한국 수도권 60%, 글로벌 테크 허브 (미국, 유럽) |
| 규모 추정 | 한국 약 50만 명, 타겟 도달 가능 약 10만 명 (출처: KOSA 2024 SW산업 실태조사 추정) |

### 행동 패턴

- GitHub 일일 활동: 커밋, 이슈 트래킹, PR 리뷰
- Hacker News, Reddit(r/LocalLLaMA, r/SelfHosted) 일일 탐색
- 한국: GeekNews, OKKY, 디스코드 개발자 서버 활동
- 새로운 AI 도구 출시 시 24시간 내 시도 (얼리 테스터)
- Product Hunt, Hacker News 런칭에 민감하게 반응
- 월 2~3개 새 도구 시험 후 1개 정도 유지

### Pain Point

1. **비용 비효율**: ChatGPT Plus $20/mo 고정 과금이지만 실 사용량은 절반 이하인 날도 多
2. **모델 락인**: 특정 LLM에 종속 — GPT만, Claude만 사용 가능한 구조
3. **개인화 부족**: 코딩 스타일, 선호 프레임워크, 프로젝트 컨텍스트를 기억하지 못함
4. **투명성 결여**: API 호출 비용, 토큰 사용량이 보이지 않음
5. **로컬 실행 제약**: 민감한 코드를 클라우드 LLM에 보내야 하는 보안 우려

### 기대 가치

- 멀티 모델 접근: 작업별 최적 LLM 자동 선택 (코딩은 Claude, 검색은 Gemini)
- 사용량 기반 과금: $5~$10/mo로 현재 대비 50~75% 비용 절감
- 개인화 메모리: 프로젝트별 컨텍스트 자동 기억 (5-Layer Memory)
- 완전한 코드 소유권: 오픈소스, 로컬 실행 가능
- 비용 투명성: 요청별 토큰 사용량/비용 실시간 확인

### 전환 트리거

| 트리거 | 설명 | 전환 확률 |
|--------|------|----------|
| 비용 절감 데모 | "월 $20 → $5~10" 실사용 비용 비교 시연 | 높음 |

## S7H-032 | HIGH | V1 | 사용자 여정 맵 (User Journey Map)

### 여정 개요

사용자 여정은 **인지 → 관심 → 설치 → 활성화 → Aha Moment → 유지 → 추천** 7단계로 구성된다.

### P1 (테크 얼리어답터) 여정

| 단계 | 활동 | 터치포인트 | 핵심 지표 | 목표 |
|------|------|-----------|----------|------|
| 1. 인지 | "VAMOS라는 AI 비서가 있대" | GitHub trending, Hacker News, GeekNews | 페이지 조회 | 월 10,000회 |
| 2. 관심 | "오, 오픈소스에 비용 투명성도?" | README, 랜딩 페이지, 기술 문서 | 클릭률 CTR | 15% |
| 3. 설치 | "설치해보자" | 원클릭 인스톨러, 5분 완료 | 설치 전환율 | 40% |
| 4. 활성화 | "첫 코딩 어시스턴트 성공!" | 온보딩 가이드, 첫 대화 | 활성화율 | 70% |
| 5. Aha Moment | "와, 내 코딩 스타일 기억하네!" | 1주 사용, 개인화 체험 | D7 리텐션 | 60% |
| 6. 유지 | "매일 코딩에 쓰게 됨" | 일일 사용, 기능 확장 | D14 리텐션 | 50% |
| 7. 추천 | "동료한테 추천" | GitHub star, 커뮤니티 공유 | NPS | 50+ |

### P2 (생산성 추구 지식 노동자) 여정

##### 행동 패턴

- 팀 SaaS 도구 다수 사용: Notion (기획), Slack (소통), Jira/Linear (PM), GitHub (개발)
- AI 비서 필요성 급증: ChatGPT Plus 개별 구독 $20/mo × 팀원 수 → 월 $60~400 부담
- 생산성 도구에 대한 평가가 빠름 (월 1~2개 도구 시험)
- 의사결정자: CEO/CTO (개인적 경험 + 비용 민감도 복합)
- 커뮤니티: 스타트업얼라이언스, 로켓펀치, YC Slack 한국인 채널

##### Pain Point

1. **복수 구독 부담**: 3명 팀 = ChatGPT Plus × 3 = $60/mo, 10명 = $200/mo (고정)
2. **개별 지식 단절**: 팀원별 ChatGPT 대화가 개별 계정에 파편화 → 팀 지식 공유 불가
3. **관리 부재**: 누가 얼마나 쓰는지, 보안 정책 통제 어려움
4. **한국 시장 AI 도구 부족**: Notion AI 등 해외 도구는 한국어 품질 + 한국 결제/세금 이슈
5. **비용 예측 어려움**: 팀원별 개별 API 키 관리 시 총 비용 가시성 부족

##### 기대 가치

- **비용 절감**: 팀원당 **$20/mo (LOCK 외 Team 티어, AUTHORITY §6 L95)** < 개별 ChatGPT Plus $20/mo × N명 = **100% 명목 동일하나 통합 관리** (STEP7-H L466 원본)
- **공유 지식 베이스**: 팀 Knowledge Graph + 공유 메모리 L3 영역 (팀 프로젝트 컨텍스트)
- **팀 Agent**: BLUE NODEs 팀 공용 커스터마이징 (예: 영업 Agent, 마케팅 Agent)
- **관리 기능**: 팀 사용량 대시보드 + 권한 관리 + 감사 로그 + SSO (Pro 플랜 부족분 Team $20/user 로 해결)
- **한국어 최적화**: 비즈니스 한국어 (기획서 / 제안서 / 이메일) 우수

```

예상 효과: 사용자 engagement +15%
```

```
