### B-1. 투자 데이터 특화 품질 규칙

| # | 항목 | 상세 |
|---|------|------|
| 1 | **가격 데이터 정합성** | OHLCV 논리적 검증: Open/Close는 High/Low 범위 내, Volume ≥ 0, 가격 제한폭 내 |
| 2 | **주식 분할/배당 조정 검증** | Split/Dividend 이벤트 전후 가격 연속성 검증. 미조정 데이터 혼입 감지 |
| 3 | **크립토 데이터 특수 검증** | 24/7 데이터 연속성, 거래소별 가격 괴리 허용 범위, 디리스팅 데이터 처리 |
| 4 | **재무 데이터 검증** | PER/PBR/ROE 논리적 범위, 전기 대비 급변 감지, 회계 기준 변경 반영 |
| 5 | **매크로 데이터 검증** | 경제 지표 발표/수정 이력 관리. 속보치(advance) vs 수정치(revised) 구분 |

### B-2. 데이터 소스 품질 관리

| # | 항목 | 상세 |
|---|------|------|
| 6 | **소스별 신뢰도 등급** | 83개 소스 각각에 대해 정확도, 지연시간, 가용성, 커버리지 등급. Tier 1(검증됨)/Tier 2(보조)/Tier 3(참고) |
| 7 | **소스 간 교차 검증** | 동일 데이터를 2개 이상 소스에서 수집 → 불일치 자동 감지. Yahoo vs KIS vs Bloomberg 가격 비교 |

| 9 | **소스 SLA(Service Level Agreement) 관리** | 각 소스의 예상 지연시간, 가용성, 데이터 범위 정의. SLA 위반 시 알림 |
