### B-1. 주문 실행 알고리즘 (Execution Algorithms)

| # | 항목 | 상세 |
|---|------|------|
| 1 | **TWAP(Time-Weighted Average Price)** | 지정 시간 동안 균등 분할 주문. 대량 매매 시 시장 충격 최소화. 주문 크기/시간 간격 자동 설정 |
| 2 | **VWAP(Volume-Weighted Average Price)** | 거래량 패턴에 맞춰 분할 주문. 장 초반/후반 거래량 높은 시간대에 더 많이 체결. 과거 거래량 프로파일 학습 |
| 3 | **IS(Implementation Shortfall) 최소화** | 의사결정 가격 vs 실제 체결 가격 차이 최소화. Almgren-Chriss 최적 실행 스케줄. 긴급도(urgency)와 시장 충격 간 트레이드오프 최적화 |
| 4 | **아이스버그(Iceberg) 주문** | 대량 주문을 소량으로 나눠 시장에 노출. 보이는 수량 제한 → 시장 충격 감소. 자동 수량 분할 및 간격 조절 |
| 5 | **적응형 실행 알고리즘** | 실시간 시장 상황(변동성, 스프레드, 유동성)에 따라 실행 알고리즘 자동 전환. 정상 시 VWAP → 급변동 시 IS 최소화 전환 |

### B-2. 스마트 주문 라우팅 (Smart Order Routing)

| # | 항목 | 상세 |
|---|------|------|
| 6 | **최적 거래소 선택 (크립토)** | 동일 코인의 Binance/Upbit/Bybit 가격 비교 → 최저 매수/최고 매도 거래소 자동 선택. 수수료 포함 총 비용 비교 |
| 7 | **호가 깊이(Order Book Depth) 분석** | 각 거래소의 호가창 깊이 실시간 비교 → 슬리피지 최소 거래소 선택 |
| 8 | **멀티 거래소 분할 체결** | 대량 주문을 복수 거래소에 분산. 한 거래소의 유동성 부족 시 자동으로 다른 거래소에 잔량 전송 |
| 9 | **한국 주식 최선 집행** | KIS API를 통한 시장가/지정가/조건부지정가 최적 선택. 장 개시 동시호가, 장 중 연속매매, 장 종료 동시호가 구분 |

### B-3. 주문 유형 & 타이밍 최적화

| # | 항목 | 상세 |
|---|------|------|
| 10 | **지정가 vs 시장가 자동 결정** | 긴급도 높음 → 시장가(확실한 체결), 긴급도 낮음 → 지정가(가격 개선). 현재 스프레드/변동성 기반 자동 결정 |
| 11 | **주문 타이밍 최적화** | 장 중 최적 시간대 분석. 스프레드가 좁고 유동성 높은 시간대 우선 체결. 장 개시 직후/마감 직전 변동성 회피 |
| 12 | **조건부 주문 체인** | "A 종목 매도 체결 → B 종목 매수" 연쇄 주문. 포트폴리오 리밸런싱 시 매도→매수 순서 최적화 |
| 13 | **크립토 24/7 타이밍** | 아시아/유럽/미국 세션별 유동성 차이 활용. 주말 저유동성 시간대 대량 주문 회피. 펀딩 레이트 정산 시점(8시간마다) 활용 |

### B-4. 슬리피지 & 시장 충격 관리

| # | 항목 | 상세 |
|---|------|------|
| 14 | **실시간 슬리피지 추정기** | 주문 전 예상 슬리피지 = f(주문 크기, 호가 깊이, 변동성, 스프레드). 예상 슬리피지 > 임계치 → 주문 지연 또는 분할 |
| 15 | **시장 충격 모델** | 일시적 충격(Temporary Impact): 주문 직후 가격 변동, 영구적 충격(Permanent Impact): 정보 효과. Almgren-Chriss 선형/비선형 모델 |
| 16 | **슬리피지 사후 분석** | 체결 후 실제 슬리피지 기록 → 모델 정확도 검증 → 모델 파라미터 지속 업데이트 |
| 17 | **유동성 부족 시 주문 보류** | 일평균 거래량 대비 주문 크기 > 5% → 자동 분할 또는 보류 경고. 비유동 종목/코인 대량 주문 방지 |

### B-5. 포지션 진입/청산 최적화

| # | 항목 | 상세 |
|---|------|------|
| 18 | **분할 진입(Scaling In)** | 시그널 발생 시 전량 진입 대신 3~5회 분할 진입. 평균 매수가 최적화 + 타이밍 리스크 분산 |
| 19 | **분할 청산(Scaling Out)** | 목표가 1/2/3 단계별 부분 익절. 수익 확보 + 추가 상승 여지 유지 동시 달성 |
| 20 | **DCA(Dollar-Cost Averaging) 엔진** | 장기 투자 시 정기 분할 매수 자동화. 매수 간격(일/주/월), 금액, 조건(하락 시 추가 매수) 설정 |
| 21 | **트레일링 스탑(Trailing Stop) 최적화** | ATR 기반 동적 트레일링 간격. 변동성 높으면 넓게, 낮으면 좁게. 이익 구간에서만 활성화 |

### B-6. 리밸런싱 최적화

| # | 항목 | 상세 |
|---|------|------|
| 22 | **리밸런싱 트리거 규칙** | 정기(월간/분기) + 비정기(비중 ±5% 이탈 시) 하이브리드. 과도한 리밸런싱(비용 낭비) vs 과소 리밸런싱(리스크 누적) 균형 |
| 23 | **세금 효율적 리밸런싱** | 매도 시 세금 영향 고려. 장기 보유(양도세 감면) vs 단기 보유 구분. Tax-Loss Harvesting 우선 매도 |
| 24 | **거래 비용 인지 리밸런싱** | 리밸런싱 이득 > 거래 비용일 때만 실행. 비중 이탈이 작으면 리밸런싱 보류(비용 대비 불리) |
| 25 | **다자산 동시 리밸런싱** | 미국 주식/한국 주식/크립토 동시 리밸런싱 시 환율, 시간대, 시장 영업시간 차이 고려. 주문 순서 최적화 |
