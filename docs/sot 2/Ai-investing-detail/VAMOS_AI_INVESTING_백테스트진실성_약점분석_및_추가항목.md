### B-3. Data Snooping & Multiple Testing 보정

**현재**: 완전히 없음
**필요한 것**:

| # | 항목 | 상세 |
|---|------|------|
| 10 | **다중 비교 보정(Multiple Testing Correction)** | 96개 전략을 동시 테스트하면 우연히 좋은 전략 발견 확률 높음. Bonferroni / Holm / BH(FDR) 보정 적용 → 조정 p-value 기준 전략 채택 |
| 11 | **White's Reality Check** | 전략이 "데이터 마이닝으로 발견된 것"인지 검증. Bootstrap으로 모든 전략의 우연한 최고 성과 분포 생성 → 실제 최고 전략이 이 분포를 유의하게 초과하는지 |
| 12 | **Hansen's SPA(Superior Predictive Ability) 테스트** | White's RC 개선판. 벤치마크 대비 통계적으로 유의한 전략만 채택 |
| 13 | **전략 개발 히스토리 기록** | 몇 개의 전략을 시도했고, 몇 개를 폐기했는지 기록. "100개 시도 중 5개 채택" → 실질 유의수준 보정 필요 |

### B-4. 과적합(Overfitting) 종합 방지

**현재**: Walk-Forward + Decay 단일 지표만 존재
**필요한 것**:

| # | 항목 | 상세 |
|---|------|------|
| 14 | **파라미터 안정성 테스트(Parameter Stability)** | 최적 파라미터 ±10~20% 변경 시에도 성과 유지되는지. 급격한 성과 변화 = 파라미터에 과적합. 3D 성과 표면 시각화 |
| 15 | **자유도(Degrees of Freedom) 제한** | 전략의 조절 가능 파라미터 수 / 데이터 포인트 수 비율 제한. 파라미터 > 데이터의 1% → 과적합 경고 |
| 16 | **교차 검증(Cross-Validation) for Time Series** | 일반 k-fold는 시계열에 부적합. Purged K-Fold CV(de Prado) 사용 → 학습/검증 사이 갭(purge) 삽입으로 정보 누출 방지 |
| 17 | **CPCV(Combinatorially Purged Cross-Validation)** | de Prado의 CPCV 기법. 모든 가능한 학습/검증 조합 생성 → 성과 분포 산출 → 분포의 중앙값으로 성과 추정 |
| 18 | **In-Sample vs Out-of-Sample 비율 규칙** | IS:OOS 최소 비율 강제(예: 3:1). OOS 기간이 너무 짧으면 검증력 부족 경고 |

**현재**: VaR 계산용 Monte Carlo만 존재
**필요한 것**:

| # | 항목 | 상세 |
|---|------|------|
| 23 | **거래 순서 셔플링(Trade Shuffling)** | 실제 거래의 순서를 무작위로 섞어 N회 시뮬레이션. 순서에 관계없이 양수 수익 → 전략의 엣지가 진짜, 순서 의존적이면 운 |
| 24 | **부트스트랩 신뢰구간** | 수익률 시계열을 블록 부트스트랩으로 리샘플링 → Sharpe/MDD의 95% 신뢰구간 산출. 구간이 넓으면 성과 불확실 |
| 25 | **랜덤 진입(Random Entry) 비교** | 전략 시그널을 무작위 진입으로 대체 → 무작위 대비 유의하게 나은지 검증. 시장 자체 상승 효과 제거 |
