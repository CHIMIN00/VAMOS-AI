## Part2 원문 (When/Where)

| 채널 | 대상 | 조건 | 버전 |
|------|------|------|------|

| Webhook (Discord/Slack) | 운영팀 | P0/P1 인시던트, 배포 실패, 비용 초과 | V2+ |

## SOT2 상세 (What/How)

### 알림 규칙 설정

| 규칙 | 조건 | 채널 | 반복 |
|------|------|------|------|
| 비용 경고 | 월 비용 ≥ 70% ($140) | Tauri + 로그 | 일 1회 |
| 비용 주의 | 월 비용 ≥ 85% ($170) | Tauri + Webhook | 12h |
| 비용 위험 | 월 비용 ≥ 95% ($190) | 전 채널 | 6h |
| 비용 초과 | 월 비용 ≥ 100% ($200) | 전 채널 + API 차단 | 즉시 |
| P0 인시던트 | 서비스 불가 감지 | 전 채널 | 5분 |
| P1 인시던트 | 핵심 기능 장애 | Webhook + 로그 | 15분 |
| 응답 시간 초과 | P95 > 10초 | Webhook | 30분 |
| 디스크 부족 | 사용률 > 85% | Tauri + 로그 | 일 1회 |

### 중복 알림 억제

- 동일 규칙 알림: 위 반복 주기 내 중복 발송 금지
- 에스컬레이션: P2 알림 후 30분 미해결 또는 영향 서비스 2개 이상 시 P1로 승격 — 수동 (당번 판단) (정본: OPERATIONS §5.3)
