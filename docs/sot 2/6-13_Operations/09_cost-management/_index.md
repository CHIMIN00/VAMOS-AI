## Part2 원문 (When/Where)

```
1. 경고(70%): 대시보드 노란색 표시, 로그 경고
2. 주의(85%): Webhook 알림 발송, 비용 높은 모듈 자동 throttle
3. 위험(95%): 사용자 알림 + P2 이상 요청 일시 정지 대기
4. 초과(100%): 모든 유료 API 호출 차단, P0/P1만 Ollama 로컬로 전환
5. 월 초 자동 리셋: daily_limit 기준 일일 배분 (LOCK)
```

## SOT2 상세 (What/How)

### 비용 추적 구현

| 항목 | 방법 |
|------|------|
| LLM API 비용 | 토큰 수 × 모델별 단가 (로그에서 추출) |
| RunPod GPU | 사용 시간 × 시간당 단가 |
| Hetzner 서버 | 고정 월 비용 (~$7.5) |
| 합계 | 실시간 누적 → SQLite/PG 메트릭 테이블 |

### Throttle 로직

```
85% 도달 시:
  - 비용 상위 모듈 식별 (COND > CORE 순)
  - COND 배치 크기 50% 감소
  - RAG 재검색 횟수 제한 (3→1회)

95% 도달 시:
  - P2/P3 요청 대기열 보류
  - P0/P1만 즉시 처리

100% 도달 시:
  - 모든 유료 API 차단
  - P0/P1 → Ollama 로컬 모델 전환
  - 사용자 알림: "월 예산 소진. 로컬 모드로 운영 중"
```

### 월 초 리셋

- daily_limit = $200 / 해당 월 일수 (예: 30일 → $6.67/일)
- 일일 미사용분 이월 허용 (단, 월간 $200 LOCK 초과 불가)
