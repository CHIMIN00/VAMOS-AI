## 7. Acceptance Criteria

- **AC-D4-001**: ToolRegistry는 D4 SOT로만 확정되며, 타 문서가 ToolRegistry의 값 목록/구조를 재정의하지 않는다.

- **AC-D4-002**: D4의 모든 스키마 블록은 D1 Meta Template 형식을 따른다(필수 _meta 포함, 신규 스키마는 ref_legacy 미사용).

- **AC-D4-003**: ToolRegistryEntrySchema.required_gates는 07 Gate(policy/cost/approval/evidence/self_check)와 정합되어야 한다.

- **AC-D4-004**: event_type / failure_code / fallback_id 값 목록은 02(ORANGE CORE) 정본이며, D4는 REF-only로만 연결한다.

- **AC-D4-005**: BrainAdapterResponseSchema.trace_id는 추적/감사 목적상 필수로 포함된다(가능하면 항상 포함).

- **AC-D4-006**: 모든 OTHER BRAINS/Tool 실행은 ToolRegistry 경유로만 수행되며, 직접 HTTP/Playwright/SDK 호출은 금지된다. (04 §4.1.1, §7.2 LOCK)

## 8. Failure / Fallback (REF-only)

> RULE (LOCK): failure_code / fallback_id 값 목록 정본은 02(ORANGE CORE) Registry이며, D4는 값을 정의/복제하지 않는다(REF-only).

### 8.1 REF 연결

- Event Registry (REF-only): 02 §6.1
- Failure Registry (REF-only): 02 §6.2
- Fallback Registry (REF-only): 02 §6.3

### 8.2 대표 시나리오 (의미/처리 방향만 서술)

| Scenario | Gate/Origin | D4 처리 | 02 Failure/Fallback 사용 |
