## 8. Failure / Fallback (REF-only)

> RULE (LOCK): failure_code / fallback_id 값 목록 정본은 02(ORANGE CORE) Registry이며, D5는 값을 정의/복제하지 않는다(REF-only).
> NOTE: 본 섹션은 05에서 LOCK로 명시된 "사용 값"과 처리 규칙만 연결한다.

### 8.1 REF 연결

- Event Registry (REF-only): 02 §6.1
- Failure Registry (REF-only): 02 §6.2
- Fallback Strategy Registry (REF-only): 02 §6.3

### 8.2 시나리오 매핑 (05 §8.2 기반, 값 생성 금지)

| failure_code (REF) | 트리거(단계) | 처리(05 LOCK) | 재시도 | fallback_id | 연동(07 Gate) |
|--------------------|-------------|---------------|--------|-------------|---------------|
| POLICY_DENY | Verify(PolicyCheck) | deny | 0 | REF-only(02 §6.3) | PolicyCheckSchema 결과 deny |
| GT_ERR_COST_LIMIT | Plan/Execute(CostBudget) | deny | 0 | REF-only(02 §6.3) | CostBudgetSchema 차단 상태 |
| TOOL_TIMEOUT | Execute(도구 호출) | retry(1회) → fallback/deny | 1 | REF-only(02 §6.3) | Gate 결과 우선(deny/hold/downshift) |

### 8.3 Soft loop / Hard loop 규칙 (05 §4.2 LOCK)

- **Soft loop**: 동일 단계 내에서 입력/프롬프트/검색 범위를 조정해 1회 보정 후 재시도. **자동 1회 허용**.
- **Hard loop**: 워크플로우를 한 단계 이전으로 되감아(또는 계획 재수립) 재실행. **승인(Approval) 필요, 자동 실행 금지**.
