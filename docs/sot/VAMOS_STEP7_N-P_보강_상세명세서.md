## Part 1 — 워크플로우 엔진 (10건)

### S7-N-001 | CRITICAL | V1 | VAMOS 워크플로우 엔진 아키텍처
- **구현 방식**: DAG(Directed Acyclic Graph) 기반 워크플로우 실행기 구현. 노드(Node)=개별 작업 단위, 엣지(Edge)=데이터 흐름+조건, 트리거(Trigger)=워크플로우 시작 조건, 변수(Variable)=워크플로우 간 데이터 전달, 에러 핸들러=실패 시 재시도/폴백/알림
- **노드 유형**: LLM Node(AI 텍스트 처리: 요약·분류·생성), Code Node(Python/JS 코드 실행), API Node(외부 API 호출), Browser Node(웹 자동화), File Node(파일 읽기/쓰기/변환), Decision Node(조건 분기), Loop Node(반복 처리), Human Node(사용자 입력 대기), Schedule Node(시간 기반 트리거), Notification Node(알림 발송)
- **기술 스택**: Python asyncio, DAG 실행 엔진, JSON 워크플로우 정의
- **VAMOS 연동**: I(Input: 사용자 워크플로우 정의) / E(Execute: 엔진 실행) / S(Store: 워크플로우 저장) / A(Automate: 자동 스케줄)
- **시중 도구 비교**: n8n(오픈소스·셀프호스팅·노드기반), Make.com(비주얼·클라우드·유료), Zapier(가장 많은 연동·유료). VAMOS 차별화 = AI 네이티브 + 대화형 생성 + 개인화 + 로컬
- **V1/V2/V3**: V1 기본 엔진 즉시 구현 | V2 비주얼 에디터 3개월

### S7-N-002 | CRITICAL | V1 | 자연어 → 워크플로우 생성
- **구현 방식**: 대화로 워크플로우 자동 생성. 예시 — User: "매일 아침 9시에 관심 종목 시세 확인하고 변동이 3% 이상이면 알려줘" → VAMOS가 자동 생성: (1) Schedule Trigger 매일 09:00 (2) API Node yfinance 관심종목 시세 조회 (3) Code Node 전일 대비 변동률 계산 (4) Decision Node 변동률 > 3%? (5) [Yes] Notification Node 알림 발송 (6) [No] 종료 → 워크플로우 JSON 자동 생성 + 미리보기 + 확인
- **수정도 대화로**: "알림을 이메일로도 보내줘" → 노드 자동 추가
- **기술 스택**: LLM 프롬프트 엔지니어링, JSON Schema, Structured Outputs
- **VAMOS 연동**: I(자연어 입력) / E(LLM 파싱+생성) / S(워크플로우 JSON 저장)
- **V1/V2/V3**: V1 즉시 (LLM 기반)

### S7-N-003 | CRITICAL | V1 | 워크플로우 트리거 시스템
- **구현 방식**: 다양한 트리거 유형 지원 — 시간 기반(cron 스케줄: 매일/매주/매월), 이벤트 기반(파일 변경·이메일 수신·웹훅), 조건 기반(주가 변동·시스템 리소스·날씨), 수동(사용자 명시적 실행), 대화 기반(특정 키워드 감지), 앰비언트(상황 감지: 위치·시간·활동)
- **트리거 조합**: AND/OR 로직 (A 이벤트 + B 조건 충족 시)
- **기술 스택**: APScheduler, watchdog, webhooks
- **VAMOS 연동**: I(트리거 조건 설정) / E(트리거 감지+실행) / A(자동 반복)
- **V1/V2/V3**: V1 시간+수동 즉시 | V2 이벤트 기반 2개월

### S7-N-004 | HIGH | V1 | 워크플로우 템플릿 라이브러리
- **구현 방식**: 기본 제공 템플릿 — 투자(일일 시장 브리핑, 관심 종목 모니터링, 공시 알림, 주간 포트폴리오 리포트, 리밸런싱 알림), 코딩(PR 자동 리뷰, 일일 코드 커밋 요약, 의존성 업데이트 알림, 에러 로그 분석), 생산성(일일 브리핑(일정+뉴스+할일), 이메일 자동 분류, 회의 노트 정리, 주간 회고), 콘텐츠(블로그 포스팅 파이프라인, SNS 자동 포스팅, 뉴스레터 생성)
- **기술 스택**: JSON/YAML 템플릿, Jinja2 변수 치환
- **VAMOS 연동**: I(템플릿 선택) / E(커스터마이징) / S(사용자 라이브러리 저장)
- **V1/V2/V3**: V1 코드 템플릿 즉시 | V2 비주얼 템플릿 2개월

### S7-N-005 | HIGH | V2 | 워크플로우 비주얼 에디터
- **구현 방식**: React Flow 기반 비주얼 편집기 — 드래그 앤 드롭 노드 배치, 노드 간 연결선 드래그, 노드 설정 패널(우측), 실시간 미리보기, 실행 상태 시각화(녹색:성공·빨강:실패), 디버그 모드(각 노드 입출력 확인)
- **n8n과 유사한 UX + VAMOS AI 통합** (자연어 수정 가능)
- **기술 스택**: React Flow, TypeScript, WebSocket 실시간 업데이트
- **VAMOS 연동**: I(비주얼 편집) / E(실행+시각화) / S(워크플로우 저장)
- **V1/V2/V3**: V2 4개월 (React Flow + 백엔드)

- **기술 스택**: Playwright, Browserbase, GPT-4V/Claude Vision

### S7-N-012 | HIGH | V1 | 웹 스크래핑 자동화
- **구현 방식**: 구조화 데이터 추출 — HTML→정형 데이터(BeautifulSoup+AI), 테이블 자동 추출→DataFrame, 가격/날짜/숫자 자동 파싱, 페이지네이션 자동 처리, 동적 콘텐츠(JavaScript 렌더링). AI 스크래핑 — "이 페이지에서 제품 이름과 가격을 추출해"→자동 셀렉터, 페이지 구조 변경 자동 적응, robots.txt 준수+Rate Limiting
- **기술 스택**: BeautifulSoup4, Playwright, pandas DataFrame
- **VAMOS 연동**: I(URL+추출 지시) / E(스크래핑 실행) / S(DataFrame 저장)

- **구현 방식**: 텍스트 감정 분석 — 기본 감정(기쁨·슬픔·분노·불안·놀람·혐오·중립), 세부 감정(피로·스트레스·좌절·열정·호기심·지루함), 한국어 감정 분석(KoBERT 기반+LLM 보조), 강도 측정(1-10 척도), 컨텍스트 인식("괜찮아"=진짜 괜찮음 vs 강한 부정). 음성 감정 분석(V2) — 톤·피치·속도·볼륨 분석, SpeechBrain/Hume AI 모델, 실시간 감정 추적. 멀티모달 감정(V3) — 텍스트+음성+표정(카메라) 통합
- **활용**: 응답 톤 자동 조절(사용자가 화났을 때 차분하게), 투자 의사결정 경고("지금 감정적인 상태에서 매매하시는 건 위험할 수 있어요"), 집중도 추정(대화 패턴으로 피로도 감지)
- **기술 스택**: KoBERT, LLM 감정 분석, SpeechBrain(V2), Hume AI(V2)
- **VAMOS 연동**: I(텍스트/음성 입력) / E(감정 분석) / S(감정 기록 저장)
- **프라이버시**: 감정 데이터 로컬 전용 처리, opt-in 명시적 동의, 감정 데이터 외부 전송 절대 없음
- **V1/V2/V3**: V1 텍스트 분석 즉시 | V2 음성 3개월

- **VAMOS 연동**: I(감정 감지) / E(적응형 응답 생성)
