## 2. REF / SEC-ID / 앵커 규칙 (LOCK)

### 2.1 REF 문법 (LOCK)

| 규칙 | 내용 |
|------|------|
| 구분자 | 콜론(:)만 사용 (파서가 단일 규칙으로만 해석 가능해야 함) |
| 앵커 slug | kebab-case를 표준으로 사용 |
| 숫자 섹션 유지 | ref_legacy로만 허용 (예: 2.1, 3.1) |

### 2.2 ref / ref_locked / ref_legacy 규칙 (LOCK)

| 유형 | 용도 | 형식 |
|------|------|------|
| ref | 특정 버전 참조 | `[REF:{DOC}:{sec_id_slug}:Schema:{SchemaName}:{schema_version}]` |
| ref_locked | 정본 포인터 (버전 고정 + 와일드카드) | `[REF:{DOC}:{sec_id_slug}:Schema:*:{schema_version}]` |
| ref_legacy | 레거시 앵커 유지 (옵션) | `[REF:{DOC}:{legacy_anchor}:{SchemaName}:{schema_version}]` |

### 2.3 SEC-ID 규칙 (LOCK)

| 규칙 | 내용 |
|------|------|
| 내부 링크/앵커 | sec_id_slug와 동일한 slug를 사용 (문서 전체 일관) |
| 명명 규칙 | kebab-case (예: `policy-check-schema`, `memory-record-schema`) |

### 2.4 ref_legacy 사용 조건 (LOCK)

- ref_legacy는 **기존 문서의 숫자 앵커를 유지해야 할 경우에만** 허용
- 신규 문서는 반드시 kebab-case slug만 사용
- ref_legacy 사용 시 Q1에 "레거시 앵커 유지 사유"를 기록해야 함

### 2.5 ID/네이밍 충돌 해결 규칙 (LOCK, REF-only)

- 정본: 01(Overview) §0.6
- 네임스페이스 충돌 해결 규칙:
  - EVX-*: Verify 확장 전용
  - E-*: 외부기능 전용
  - S-#: Self-evo 모듈 ID
  - S#_...: State machine 상태명
  - slug 표준: kebab-case

### 3.1 필수 _meta 블록

D# 문서에 들어가는 모든 스키마는 아래 `_meta`를 반드시 포함한다.

- 규칙(LOCK): `schema_version/ref/ref_locked`는 문서 버전 승격(v3.0.0)에 맞춰 동일 버전을 사용한다.
- 규칙(LOCK): `ref_legacy`는 **레거시 앵커 유지가 필요한 경우에만** 사용하며, 신규 스키마는 **키 자체를 제거**한다.

```json
{
  "_meta": {
    "schema_name": "XxxSchema",
    "schema_version": "v3.0.0",
    "owner_doc": "D#",
    "sec_id_slug": "kebab-case-slug",
    "ref": "[REF:D#:kebab-case-slug:Schema:XxxSchema:v3.0.0]",
    "ref_locked": "[REF:D#:kebab-case-slug:Schema:*:v3.0.0]"
  },
  "description": "…(1~2줄, 목적/범위만)",
  "fields": [
    {
      "name": "field_name",
      "type": "string|number|boolean|object|array",
      "required": true,
      "example": "example_value",
      "description": "필드 설명"
    }
  ]
}
```

### 3.2 필드 규칙 (LOCK)

| 규칙 | 내용 |
|------|------|
| 필수 속성 | name / type / required / example / description |
| enum 필드 | 반드시 레지스트리 값(SOT)과 연결, 임의 값 생성 금지 |
| 버전 형식 | `v{major}.{minor}.{patch}` (예: v3.0.0) |
