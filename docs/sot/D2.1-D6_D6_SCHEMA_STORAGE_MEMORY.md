## 6. Acceptance Criteria

- **AC-D6-001**: D6의 모든 스키마 블록은 D1 Schema Meta Template의 `_meta` 필수 속성(`schema_name`, `schema_version`, `owner_doc`, `sec_id_slug`, `ref`, `ref_locked`, `ref_legacy`)을 포함한다.

- **AC-D6-002**: MemoryRecordSchema.scope enum은 `L0 | L1 | L2 | L3`만 허용한다 (06 §2.1 LOCK).

- **AC-D6-003**: MemoryRecordSchema.memory_type enum은 `B-1 | B-2 | B-3 | B-4`만 허용한다 (06 §2.1 LOCK).

- **AC-D6-004**: MemoryRecordSchema.policy_decision enum은 `allow | restrict | deny`만 허용하며, D7 PolicyCheckSchema.decision과 정합한다.

- **AC-D6-005**: §5에 Example JSON이 최소 MemoryRecord 2개(L1/B-1, L3/B-2) + SourceQoD 1개 존재한다.

- **AC-D6-006**: L3/B-2 절차 메모리의 추가 필드(`procedure_id` ~ `provenance`)는 "확장 필드(선택)"로 제공되며, 운영 규칙상 L3/B-2에서 확보 대상이다 (06 §2.4.2).

- **AC-D6-007**: Failure/Fallback에서 event_type/failure_code/fallback_id의 값 목록은 02 정본을 REF-only로 참조하며, D6에 목록을 박제하지 않는다.

- **AC-D6-008**: VectorStoreAdapterSchema.backend 전환(chroma→qdrant)은 어댑터 교체로만 수행하며, 상위 인터페이스 변경 없이 투명 처리한다.

- **AC-D6-009**: GraphRAGConfigSchema.scope는 V1에서 P1-SCOPE (간이), V2+에서 FULL로 확장한다. (STEP7 R4, DEC-004)

- **AC-D6-010**: SemanticCacheSchema.similarity_threshold는 0.95 이상만 캐시 히트로 판정한다 (LOCK). (06 §4.7 ADD-012)

## 7. Failure / Fallback (REF-only 연결)

### 7.1 원칙
- event_type / failure_code / fallback_id의 **값 목록 정본(SOT)**은 02(Registry)이다.
- D6는 저장 도메인에서의 "의미/발생 조건/연결 흐름"만 기술하고, 정본에 없는 코드값을 확정(박제)하지 않는다.

### 7.2 저장 정책/차단 시나리오 연결

| 시나리오 | 결과 | 코드값 참조 |
|----------|------|-------------|
| PII 탐지 (storage.policy.checked 단계) | MemoryRecord 커밋 중단 또는 content_summary 최소화 + policy_decision=deny/restrict | failure_code/fallback_id → 02 정본 (REF-only) |
| 벡터 삽입 거부 (storage.vector.denied) | 벡터 저장 경로만 거부, MemoryRecord 자체는 정책 허용 범위에서 커밋 가능 | failure_code/fallback_id → 02 정본 (REF-only) |
| 장기 저장 거부 (L1/L2) | L1/L2 커밋 불가 처리(deny/restrict) + 필요 시 D7 Approval Gate 연동 | failure_code/fallback_id → 02 정본 (REF-only) |
| 마스킹 불완전 | 보수 처리(deny 또는 restrict + 승인 전 HOLD), 원문 저장 금지 유지 | failure_code/fallback_id → 02 정본 (REF-only) |

### 7.3 D7 연결
- 저장 관련 deny/restrict/allow 결과는 D7 PolicyCheckSchema 결과와 정합되어야 한다.
