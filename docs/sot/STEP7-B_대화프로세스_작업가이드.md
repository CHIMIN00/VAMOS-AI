## 1. Claude (Anthropic) — 8단계 에이전트 루프

``
[SessionStart Hook 실행]
      ↓
[시스템 프롬프트 로딩]
 ├─ 메인 프롬프트 (~40 system reminders)
 ├─ CLAUDE.md (프로젝트 지침)
 ├─ 18개 도구 정의 로딩
 └─ Magic Docs (자동 문서 인식)
      ↓
[사용자 입력 수신]
      ↓
[UserPromptSubmit Hook]
      ↓
[의도 분석 + Extended Thinking]
 ├─ Adaptive Thinking (난이도 자동 판단)
 │   ├─ low → 즉시 응답
 │   ├─ medium → 짧은 사고
 │   ├─ high → 깊은 사고
 │   └─ max → 최대 사고 (연구 수준)
 ├─ thinking 블록 생성 (사용자에게 표시 가능)
 └─ 계획 수립 (Plan Mode 여부 결정)
      ↓
[권한/모드 확인]
 ├─ Normal → 사용자 승인 요청
 ├─ Auto-Accept → 자동 허용
 ├─ Plan Mode → 읽기 전용, 코드 수정 불가
 ├─ Don't Ask → 자동 실행
 └─ Bypass → 샌드박스 무시 (위험)
      ↓
[도구 사용 루프 시작] ◄──────────────────────┐
 │                                             │
 ├─ [PreToolUse Hook] ← 도구 실행 전 검증      │
 │                                             │
 ├─ [도구 선택 + 실행]                          │
 │   ├─ Read (파일 읽기)                        │
 │   ├─ Write (파일 쓰기)                       │
 │   ├─ Edit (파일 수정)                        │
 │   ├─ Bash (명령 실행)                        │
 │   ├─ Glob (파일 검색)                        │
 │   ├─ Grep (내용 검색)                        │
 │   ├─ WebSearch (웹 검색)                     │
 │   ├─ WebFetch (웹 페이지 가져오기)            │
 │   ├─ Task (서브에이전트 스폰)                 │
 │   │   ├─ Plan Agent (계획 설계)              │
 │   │   ├─ Explore Agent (코드베이스 탐색)      │
 │   │   ├─ Bash Agent (명령 실행 전문)          │
 │   │   └─ General Agent (범용 리서치)          │
 │   ├─ TodoWrite (작업 목록 관리)               │
 │   ├─ AskUserQuestion (사용자 질문)            │
 │   ├─ EnterPlanMode (계획 모드 진입)           │
 │   ├─ ExitPlanMode (계획 승인 요청)            │
 │   ├─ Skill (스킬 실행)                       │
 │   ├─ NotebookEdit (Jupyter 편집)             │
 │   ├─ TaskOutput (백그라운드 작업 결과)         │
 │   └─ TaskStop (백그라운드 작업 중지)           │
 │                                             │
 ├─ [PostToolUse Hook] ← 도구 실행 후 처리      │
 │                                             │
 ├─ [도구 결과 분석]                            │
 │   ├─ 추가 도구 필요? ─── YES ───────────────┘
 │   └─ NO → 응답 생성 진행
 │
 ↓
[컨텍스트 윈도우 관리]
 ├─ 한계 접근 시 → [PreCompact Hook]
 ├─ 이전 메시지 자동 압축
 ├─ [PostCompaction Hook]
 └─ SessionStart 재실행 (source="compact")
      ↓
[응답 생성]
      ↓
[Stop Hook 실행]
      ↓
[사용자에게 출력]
```

### VAMOS 적용 상태:

| Claude 프로세스 단계 | VAMOS 상태 | 비고 |
|---------------------|-----------|------|
| SessionStart Hook | ❌ 미적용 | Hooks 시스템 자체가 없음 |

| 서브에이전트 스폰 | ⚠️ 부분 | Agent Pool 설계만, Task 패턴 없음 |
| TodoWrite (작업 관리) | ❌ 미적용 | 에이전트 내부 작업 추적 없음 |
| AskUserQuestion | ❌ 미적용 | 명시적 사용자 질문 도구 없음 |
| Plan Mode | ❌ 미적용 | 계획 승인 모드 없음 |
| 컨텍스트 압축 | ❌ 미적용 | 메시지 자동 압축 없음 |

# 2부. VAMOS AI 현재 대화 프로세스 vs 시중 AI

## VAMOS 현재 파이프라인 (7단계 + 5단계 + 8-State Machine)

```
[사용자 입력 (텍스트/이미지/오디오/비디오/PDF)]
      ↓
[1단계: 입력 (Input)]
 ├─ Front Mini LLM
 │   ├─ MIME/Size/Duration 메타데이터 추출
 │   ├─ 1차 안전 검사 (PII 감지, 악성 스캔) → I-17
 │   ├─ 초기 전처리 (OCR/STT) → I-4
 │   └─ 도메인 힌트 생성 → I-1 입력
 └─ S0_RECEIVED 상태
      ↓
[2단계: 분석 (Analysis)]
 ├─ I-1 (대화 이해 엔진)
 │   ├─ 의도(intent) 추출
 │   ├─ 도메인/테마 분류
 │   ├─ 제약 조건 추출
 │   └─ IntentFrame 스키마 생성
 ├─ I-7 (안전 엔진) 위험 판단
 ├─ I-8 (비용 엔진) 비용 추정
 ├─ I-3 (메모리 엔진) 메모리 매핑
 └─ S1_INTENT_PARSED 상태
      ↓
[3단계: 지식·근거 검색 (Retrieval)]
 ├─ I-2 (RAG 검색 엔진)
 │   ├─ 벡터 검색 (Chroma/Qdrant)
 │   ├─ BM25 어휘 검색
 │   ├─ GraphRAG (지식 그래프)
 │   └─ 앙상블 재순위화
 ├─ I-19 (QoD 평가) 신뢰도 점수
 │   ├─ QoD ≥ 0.70 → 통과
 │   ├─ 0.70 > QoD ≥ 0.50 → 재검색
 │   └─ QoD < 0.50 → 거부/에스컬레이션
 ├─ 프로젝트/글로벌 메모리 결합
 └─ S2_EVIDENCE_READY 상태
      ↓
[4단계: 라우팅·프로세스 생성 (Routing)]
 ├─ I-5 (라우팅/결정 엔진)
 │   ├─ ★ 3-Gate 검증 ★
 │   │   ├─ Policy Gate: deny|restrict|allow
 │   │   ├─ Cost Gate: normal|downshift|split|stop
 │   │   └─ Evidence Gate: QoD 충족 여부
 │   ├─ BLUE NODE 선택 (Dev/Research/Content/Quant/Trading)
 │   ├─ 실행 모드 결정 (mini|main|tool)
 │   └─ Decision Object 생성 (locked=true)
 ├─ I-13 (템플릿 엔진) 워크플로우 생성
 └─ S3_DECISION_LOCKED 상태 ← ★ 이후 결론 변경 불가
      ↓
[5단계: 모델 실행 (Execution)]
 ├─ BLUE NODE 실행
 │   ├─ Front Mini → Main → Tool 모델 순서
 │   └─ MCP 표준 도구 호출 (@mcp.tool)
 ├─ ToolRegistry를 통한 외부 도구 실행
 │   ├─ mcp.search.tavily (웹 검색)
 │   ├─ mcp.code.e2b (코드 실행)
 │   ├─ mcp.doc.unstructured (문서 파싱)
 │   └─ mcp.browser.playwright (브라우저)
 ├─ I-8 비용 실시간 모니터링
 └─ S4_EXECUTING 상태
      ↓
[6단계: 자기검증 (Self-Check)]
 ├─ I-6 (자기검증 엔진)
 │   ├─ 논리/근거 검증
 │   ├─ 환각/오류/위험 감지
 │   ├─ Soft Check (1회 자동)
 │   └─ Hard Check (사용자 승인 필요) 조건
 ├─ 실패 시 → 재검색/재계획/재생성/거부
 ├─ S5_OUTPUT_READY → S6_SELF_CHECKED
 └─ EVX-1~EVX-6 검증 체인
      ↓
[7단계: 출력 + 저장 (Output)]
 ├─ Markdown/테이블/보고서 형식화
 ├─ 3-Part 출력: user_response + evidence_summary + log_report
 ├─ 저장 후보 판단 (I-3)
 │   ├─ L0 (세션/7일) → L1 (프로젝트/90일) → L2 (장기/영구)
 │   └─ 사용자 확인 후 저장 (PLAN 2.0 정책)
 ├─ S7_MEMORY_COMMITTED → S8_DONE
 └─ 이벤트 로깅 (oc.*.* 이벤트)
```

## 감정 분석 기술 상세

### GPT-4.5의 인지적 공감 (Cognitive Empathy)
- **방식**: 대규모 비지도 사전학습으로 "세계 모델" 구축 → 감정 민감 경로가 추론에 자연 포함
- **구현**: 별도 모듈 아님, RLHF로 강화된 이머전트 능력
- **비용**: $75/M 토큰 (GPT-4o 대비 30배 비쌈)
- **벤치마크**: 설득력 테스트 50%+ (GPT-4o는 1% 미만), 환각률 37.1% (GPT-4o 61.8%)

### Claude의 정서적 공감 (Affective Empathy)
- **방식**: 활성화 조향(activation steering)으로 감정적 반응 차원에 편향
- **구현**: Constitutional AI 원칙으로 감정 상황 핸들링 가이드
- **Hume AI 연동**: EVI(Empathic Voice Interface) 기본 LLM으로 Claude 사용, 36% 사용자 선택

| 8 | **IntentFrame 스키마** | 구조화된 의도 표현 — 시중 AI는 암묵적 |
| 9 | **Fallback Chain** | 실패 시 단계별 대안 (재검색→모델 다운시프트→출력 제한→거부) |

| 11 | **비용 실시간 모니터링** | 실행 중 비용 추적 + 초과 시 자동 차단 |

## 추가 감정 AI 시스템

### Sesame CSM (Conversational Speech Model, 2025.03)
- **핵심**: 세계 최초로 진정한 자연스러운 음성 대화 가능
- **방식**: 오디오 코덱 토큰 + 텍스트 토큰 동시 생성. 대화 맥락을 이해하는 음성 모델
- **감정**: 음성의 미세한 감정 뉘앙스(주저함, 흥분, 차분함)를 자연스럽게 재현
- **구현성**: 오픈소스 (PyTorch), V3에서 로컬 실행 가능 (GPU 필요)
- **VAMOS 적용**: V3 TTS 엔진 후보. 기존 ElevenLabs/Google TTS보다 자연스러운 대안

### Inflection Pi 3 감정 AI
- **핵심**: "가장 공감적인 AI" 목표. 감정적 지지, 대화 치료 수준 응답
- **방식**: 대규모 감정 대화 데이터로 학습. 감정 인식이 아닌 감정적 응답 생성에 집중
- **VAMOS 적용**: ResponseAdaptation 모듈에서 Pi 스타일 공감 응답 템플릿 참조

### 문화별 감정 차이 (VAMOS 필수)
- 한국: 간접 표현, "괜찮아요"가 실제로 괜찮지 않을 수 있음. 사회적 맥락 중요
- 일본: 혼네(본심)/다테마에(겉모습) 구분 필요. 표면적 표현 ≠ 실제 감정
- 영어권: 비교적 직접적 감정 표현
- **구현**: EmotionFrame의 cultural_context 필드를 확장하여 문화별 감정 해석 규칙 적용
