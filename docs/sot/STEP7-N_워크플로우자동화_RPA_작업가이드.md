## Part 1: 워크플로우 엔진 [10항목]

### N-001. VAMOS 워크플로우 엔진 아키텍처
```
[구현 상세]
- DAG 기반 워크플로우 실행기:
  ├─ 노드(Node): 개별 작업 단위
  ├─ 엣지(Edge): 데이터 흐름 + 조건
  ├─ 트리거(Trigger): 워크플로우 시작 조건
  ├─ 변수(Variable): 워크플로우 간 데이터 전달
  └─ 에러 핸들러: 실패 시 재시도/폴백/알림

- 노드 유형:
  ├─ LLM Node: AI 텍스트 처리 (요약, 분류, 생성)
  ├─ Code Node: Python/JS 코드 실행
  ├─ API Node: 외부 API 호출
  ├─ Browser Node: 웹 자동화
  ├─ File Node: 파일 읽기/쓰기/변환
  ├─ Decision Node: 조건 분기
  ├─ Loop Node: 반복 처리
  ├─ Human Node: 사용자 입력 대기
  ├─ Schedule Node: 시간 기반 트리거
  └─ Notification Node: 알림 발송

[시중 도구 비교]
- n8n: 오픈소스, 셀프호스팅, 노드 기반
- Make.com: 비주얼, 클라우드, 유료
- Zapier: 가장 많은 연동, 유료
- VAMOS 차별화: AI 네이티브 + 대화형 생성 + 개인화 + 로컬

[구현성] V1: ✅ 기본 엔진 즉시 | V2: ✅ 비주얼 에디터 3개월
```

### N-019. ETL 자동 파이프라인
```
[구현 상세]

  ├─ Extract: API, 웹, 파일, DB에서 데이터 수집
  ├─ Transform: 정제, 변환, 집계, 필터링
  ├─ Load: DB, 파일, 대시보드에 적재

- 자연어 → ETL:
  "매일 DART에서 관심 종목 공시를 가져와서
   종목명, 공시제목, 날짜를 추출하고
   투자 노트에 저장해줘"

[구현성] V1: ✅ 즉시 (pandas + 스케줄러)
```
