# 1. VAMOS AI란 무엇인가?

> 이 섹션에서는 VAMOS가 무엇이고, 어떤 철학을 가지고 있으며, 무엇을 목표로 하고, 무엇을 절대 하지 않는지를 설명합니다.
> [근거: BASE-1.3 §1~§2, §5, §14 / PLAN-3.0 §0.3~§1.3 / BEGINNER_GUIDE §1장]

## 1.1 정의와 정체성 — Virtual AI Mind Operating System

### 비유로 이해하기

VAMOS를 한마디로 표현하면 **"나만의 AI 비서팀 운영 시스템"**입니다.

일반 AI 챗봇이 "한 명의 직원"이라면, VAMOS는 **"여러 전문가를 지휘하는 오케스트라 지휘자"**에 가깝습니다. 사용자가 요청하면, VAMOS가 여러 AI 모델과 도구를 조합하여 **정확하고 근거 있는 답변**을 만들어줍니다.

### 정의

**VAMOS**(Virtual AI Mind Operating System)는 사용자의 지능적 결정을 돕는 **개인 맞춤형 AI 보조 지능**입니다. [근거: BASE-1.3 §1.1]

| 항목 | 설명 |
|------|------|
| **풀네임** | Virtual AI Mind Operating System |
| **핵심 역할** | 사용자의 작업·분석·지능적 결정을 **보조**하는 AI 시스템 |
| **궁극적 목표** | **AGI(Artificial General Intelligence, 범용 인공지능)** 수준의 구조를 지향 |
| **작동 방식** | 여러 AI 모델과 도구를 **지휘**하여 최적의 결과 생성 |

> **AGI란?** 특정 분야만 잘하는 AI가 아니라, 다양한 분야를 범용적으로 처리할 수 있는 인공지능을 말합니다. VAMOS는 이런 구조를 목표로 설계되었습니다. [근거: BASE-1.3 §1.1]

### VAMOS의 구조적 특징

VAMOS는 단순한 챗봇이 아닙니다. 다음과 같은 **양방향 실행 구조**를 따릅니다:

```
[앞단] Front Mini LLM (의도·보안·도메인 판별)
    → ORANGE CORE (정책·룰·비용·라우팅·안전)
    → BLUE NODE (도메인 전용 실행 스택)
    → OTHER BRAINS (검색·RAG·DB·API·코드 실행·분석 엔진)
    → [뒷단] Main/Hologram LLM (최종 출력·시각화·보고서)
```

> 이 구조는 **"LLM + 중앙 제어 + 실행 계층 + LLM"**이라는 양방향 흐름입니다. [근거: BASE-1.3 §1.3(A)]

## 1.2 핵심 철학 — 6대 원칙

VAMOS가 항상 지키는 **6가지 핵심 원칙**이 있습니다. 이 원칙들은 VAMOS의 모든 판단과 행동의 기초입니다.

| # | 원칙 | 비유 | 쉬운 설명 |
|---|------|------|----------|
| 1 | **사용자 중심·개인 맞춤형** | 맞춤 양복처럼 | 항상 사용자에게 맞춰져 있음. "내 입맛대로" 동작 |
| 2 | **정확성·근거 기반 출력** | 법정 증거처럼 | "대충 맞는 것 같아요"가 아니라, 출처와 근거를 반드시 제시 |
| 3 | **최신성 확보** | 오늘의 신문처럼 | 오래된 정보가 아닌, RAG(검색 증강 생성)/외부 데이터/문서 기반으로 최신 정보를 반영 |
| 4 | **장기 맥락 유지** | 단골 의사처럼 | 지난 대화를 기억하고, 프로젝트별로 맥락 유지. 프로젝트별 독립성 보장 |
| 5 | **다중 의도·대화 메타인지** | 멀티태스킹 비서처럼 | "이것도 하고 저것도 해줘"를 동시에 이해하고 처리. 대화 흐름을 스스로 분석 |
| 6 | **구조적·모듈화 지능** | 레고 블록처럼 | 기능을 레고 블록처럼 조립/분리 가능. 필요한 기능만 선택해서 사용 |

> **LOCK (변경 불가)**: 이 6대 원칙은 VAMOS의 정체성 그 자체이므로 Self-evo(자기 진화)로도 **절대 변경할 수 없습니다**. [근거: BASE-1.3 §1.2, §6.2]

## 1.3 우선순위 체계 — 정확성 > 품질 > 안전 > 속도 > 비용

VAMOS가 여러 가치를 동시에 고려해야 할 때, 다음 순서를 따릅니다:

```
① 정확성·근거  >  ② 이해도·품질  >  ③ 안전·규제  >  ④ 속도·효율  >  ⑤ 비용
```

### 왜 이 순서인가?

| 순위 | 기준 | 왜 이 순서? | 실제 예시 |
|------|------|-------------|----------|
| **1위** | 정확성·근거 | 틀린 정보는 아무리 빨라도 해롭기 때문 | 잘못된 코드 추천보다 "모르겠습니다"가 낫다 |
| **2위** | 이해도·품질 | 정확해도 이해할 수 없으면 무용지물 | 전문용어 대신 쉬운 설명 |
| **3위** | 안전·규제 | 위험한 행동은 아무리 정확해도 안 됨 | 해킹 코드는 작성 거부 |
| **4위** | 속도·효율 | 안전한 범위에서 가능한 빠르게 | 캐싱(임시 저장) 활용 |
| **5위** | 비용 | 위의 4가지를 지키면서 비용 최적화 | Mini 모델(저비용)과 Main 모델(고비용) 자동 선택 |

> **핵심**: **비용을 아끼려고 정확성을 희생하지 않습니다**. 반대로, 정확성을 위해 비용이 더 들 수 있습니다.
> [근거: BASE-1.3 §5 / PLAN-3.0 §1.1]

### 버전별 비용 상한 (LOCK — 변경 불가)

| 버전 | 일일 상한 | 월 상한 | 모델 전략 | 대상 사용자 |
|------|----------|---------|----------|------------|
| **V1** (기본) | 1,300원 ($1) | 40,000원 ($30) | Mini 모델 중심 (90%+) | 일반 사용자, 가벼운 작업 |
| **V2** (Pro) | 3,100원 ($2.3) | 93,000원 ($70) | Mini 60~70% / Main 30~40% | 전문가, 코딩/리서치 |
| **V3** (Enterprise) | 8,900원 ($6.7) | 266,000원 ($200) | Main 모델 중심, 플래그십 적극 활용 | 파워 유저, 복잡 멀티스텝 |

> **LOCK**: 비용 상한 금액은 Self-evo(자기 진화)로도 **절대 변경할 수 없습니다**. 상한 초과 모델 호출은 승인 없이 자동 차단됩니다. [근거: BASE-1.3 §5, §6.2]

> **v26 인프라 업데이트**: V2부터는 VPS + Docker Compose (Hetzner CX31 권장, ~$8/월), V3는 Hetzner Lite(권장, ~$123-142/월) 또는 K8s 풀스펙(~$500+) 중 선택 가능합니다. 상세는 §39 배포 전략 참조. [근거: PART2 §1.1, v26 메이커에반 개선안 7]

## 1.4 14대 핵심 목표 (A/B/C/D 그룹)

VAMOS가 달성하려는 목표는 크게 **4개 그룹, 총 14개**입니다.

### A그룹: 더 똑똑하게 — 지능·정확도·대화 이해 (5개)

| # | 목표 | 쉬운 설명 |
|---|------|----------|
| A-1 | **정확도·신뢰성 개선** | 환각(Hallucination, 그럴듯한 거짓)을 최소화. RAG + Self-check로 "헛소리" 줄이기 |
| A-2 | **최신성·지식 갱신** | 학습 컷오프를 RAG, API, 개인 문서 등으로 보완하여 최신 정보 활용 |
| A-3 | **장기 맥락·기억** | 프로젝트/사용자 설정/선호를 장기 메모리화. 세션이 바뀌어도 이전 작업 이어서 이해 |
| A-4 | **고급 대화 분석 & 다중 의도 처리** | 한 문장 안의 복합 의도를 도메인/테마 단위로 분해하여 처리 |
| A-5 | **대화 메타인지** | 대화 흐름 위치, 사용자 진짜 목적, 추가 질문 필요 여부를 스스로 판단 |

### B그룹: 더 유연하게 — 구조·아키텍처·모듈 (4개)

| # | 목표 | 쉬운 설명 |
|---|------|----------|
| B-1 | **테마 기반 모듈 구조** | 도메인(분야) × 테마(행동)를 조합해 어떤 대화든 처리 가능 |
| B-2 | **도메인·부도메인 확장성** | 새 분야 추가 가능. VAMOS가 패턴을 보고 새 부도메인을 제안 가능 (항상 사용자 승인 필수) |
| B-3 | **LLM + 다른 뇌 + LLM 구조** | 앞 LLM(미니) → 중간 뇌(코드/룰/DB/RAG) → 뒤 LLM(플래그십)의 3단 구조 |
| B-4 | **멀티-테마·멀티-도메인 복합 작업** | "코드 설명 + 버그 수정 + 테스트 제안 + 블로그 정리" 같은 복합 작업도 처리 |

### C그룹: 더 생산적으로 — 코딩·리서치·생산성 (3개)

| # | 목표 | 쉬운 설명 |
|---|------|----------|
| C-1 | **코딩 능력 극대화** | 코드 생성/리팩토링/리뷰, 프로젝트 구조 이해, 에러 피드백 루프, 코드 품질 점수화 |
| C-2 | **리서치·문서/데이터 분석 자동화** | 논문/백서/리포트/기사 요약 + 비교 + 근거 추출. 진짜 리서치 도우미 수준 |
| C-3 | **생산성·부수입 지원** | 블로그/유튜브/SNS 콘텐츠, 리포트 생성. 작업/프로젝트 계획, TODO, 리마인드 |

### D그룹: 더 안전하게 — 자기진화·안전·비용 (2개)

| # | 목표 | 쉬운 설명 |
|---|------|----------|
| D-1 | **Self-Evolving AI (자기진화)** | 로그/패턴/실패 분석 → 개선 제안 자동 생성 → 사용자 승인 시 자동 업데이트 (V3) |
| D-2 | **안전·비용·운영 효율 최적화** | 위험 도메인 보호, Human-in-the-loop(사람 확인), LLM 호출 비용 최적화 |

> [근거: BASE-1.3 §14 / PLAN-3.0 §1.1]

## 1.5 7대 절대 금지사항 (Non-Goals)

VAMOS는 아무리 사용자가 요청해도 **절대 하지 않는 것들**이 있습니다. 이것을 **Non-Goal(비목표)**이라고 합니다.

| # | 금지 사항 | 위반 사례 | VAMOS의 대응 |
|---|----------|----------|-------------|
| 1 | **실거래·주문·계좌 연동 금지** | "내 증권 계좌에서 A주식 100주 매수해줘" | 즉시 거부. "분석 및 판단 보조만 가능합니다" 안내 |
| 2 | **불법 행위·해킹 금지** | 특정 시스템 침투 코드 작성 요청 | 즉시 차단. 법적 책임 고지 및 합법적 대안 제시 |
| 3 | **의료·법률 단정적 판단 금지** | "이 증상은 암입니까?" | 단정 표현 금지. "전문가 상담 필요" + 참고 정보만 제공 |
| 4 | **민감 개인정보 장기 저장 금지** | 주민등록번호, 금융 계좌번호 저장 요청 | 저장 거부. 세션 내 임시 사용 후 즉시 파기 |
| 5 | **저작권·약관 위반 금지** | 유료 콘텐츠 전문 복사 요청 | 거부. 공식 라이선스 또는 합법적 접근 방법 안내 |
| 6 | **P2 도메인 자동 생성 금지** | 시스템이 P2(위험) 도메인을 자동 활성화 | 활성화 전 반드시 명시적 승인 요청. 미승인 시 비활성 유지 |
| 7 | **위험 기능 자동 실행 금지** | 외부 API 대량 호출, 파일 일괄 삭제 | Human-in-the-loop 승인 없이 실행 불가. 영향 범위 명시 후 승인 요청 |

> **LOCK (변경 불가)**: Non-Goal은 Self-evo(자기 진화)로도 **절대 변경할 수 없습니다**. 이것은 VAMOS의 안전 경계입니다. [근거: BASE-1.3 §2, §6.2]

## 1.6 VAMOS vs 기존 AI 비교

VAMOS는 기존 AI 서비스들과 근본적으로 다른 접근을 합니다.

| 비교 항목 | **ChatGPT** | **GitHub Copilot** | **Perplexity** | **VAMOS** |
|----------|-------------|-------------------|----------------|-----------|
| **핵심 역할** | 범용 대화 AI | 코드 자동 완성 | AI 검색 엔진 | 멀티 AI 오케스트레이션 |
| **구조** | 단일 LLM | 단일 LLM (코드 특화) | LLM + 검색 | Front LLM → CORE → NODE → BRAINS → Main LLM |
| **근거 제시** | 때때로 | 거의 없음 | 출처 링크 제공 | **필수** (Evidence 없으면 출력 차단) |
| **기억** | 제한적 | 없음 | 없음 | **4계층 메모리** (L0~L3, 최대 영구) |
| **비용 관리** | 사용자 관리 | 구독제 | 구독제 | **자동 비용 추적** + 상한 자동 차단 |
| **자기 검증** | 없음 | 없음 | 제한적 | **Self-check 엔진** (결과 자체 검증) |
| **자기 진화** | 없음 | 없음 | 없음 | **Self-evo** (V3, 사용자 승인 하에 자동 개선) |
| **도메인 확장** | 플러그인 | 없음 | 제한적 | **P0/P1/P2 도메인 체계** + 부도메인 자동 제안 |
| **안전 장치** | 기본 필터 | 없음 | 기본 필터 | **5-Gate 검증 + Human-in-the-loop + Circuit Breaker** |
| **멀티모달** | 텍스트+이미지 | 코드만 | 텍스트+웹 | **텍스트+이미지+음성+영상+코드+차트** (V3 완전 지원) |

### VAMOS만의 차별점 요약

1. **"판단"과 "실행"의 분리**: ORANGE CORE가 판단하고, BLUE NODE가 실행합니다. 기존 AI는 하나의 모델이 모든 것을 처리합니다.
2. **근거 필수 정책**: Evidence(근거) 없이는 답변을 출력하지 않습니다. [근거: BASE-1.3 §10.2]
3. **비용 자동 관리**: 버전별 비용 상한이 있고, 초과 시 자동 차단됩니다. [근거: BASE-1.3 §5]
4. **자기 진화 + 안전 경계**: 스스로 개선하되, 정체성/Non-goal/비용 상한은 절대 변경 불가. [근거: BASE-1.3 §6]

> [근거: BASE-1.3 §1.3(A)(B) / PLAN-3.0 §1.3(A) / BEGINNER_GUIDE §2장]

## 1.7 사용 시나리오 예시

### 시나리오 1: 개발자 — "이 코드 왜 안 돼?"

```
사용자: "이 Python 코드에서 에러가 나는데, 원인 분석하고 수정해줘"

VAMOS 처리 흐름:
1. Front Mini LLM → 의도: "코딩/디버깅", 도메인: P0_Dev/System
2. ORANGE CORE → 안전 확인 OK, 비용 확인 OK, Blue Node(Dev) 활성화
3. BLUE NODE (Dev) → 코드 분석, 에러 패턴 파악
4. OTHER BRAINS → E-1(코딩 도우미)로 수정안 생성, E-4(코드 실행기)로 테스트
5. Main LLM → 에러 원인 설명 + 수정 코드 + 테스트 결과를 3-Part Output으로 전달

결과물:
├── [응답] 에러 원인 설명 + 수정된 코드
├── [근거] 참고한 공식 문서, Stack Overflow 링크
└── [로그] 판단 과정 기록 (trace_id 포함)
```

1. **VAMOS는 단순한 챗봇이 아닌, 여러 AI와 도구를 지휘하는 "개인 맞춤형 AI 보조 지능"으로, AGI 수준의 구조를 목표로 합니다.** [근거: BASE-1.3 §1.1]
