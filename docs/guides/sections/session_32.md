## §44.1 Cat-A: 기초 강화 — 경쟁 분석 & 차별화 혁신 (Competitive Analysis & Innovation)

### 비유

> **10개 레스토랑의 메뉴를 전부 분석**한 뒤, 각각의 인기 요리 비법을 연구하고, 거기에 **우리만의 시그니처 메뉴**를 추가하는 과정입니다.

### 항목 수: S7-A-001 ~ S7-A-316 (316개)

### 핵심 내용 요약

Cat-A는 STEP7에서 가장 큰 카테고리로, **10개 주요 AI 서비스**(Claude, GPT, Gemini, Kimo, Grok, Mistral, Llama, Perplexity, Apple Intelligence 등)와의 기술 격차를 분석하고 해소합니다. 단순 모방이 아니라 각 서비스의 강점을 VAMOS 방식으로 재해석하며, 어떤 경쟁사에도 없는 **99개 혁신 기술**(Innovation)을 포함합니다.

- 16개 세부 파트: Part A(Claude 35개), Part B(GPT 24개), Part C(Gemini 20개), Part D(Kimo 22개), Part E(VAMOS 고유 강점 18개), Part F(혁신 99개), Part G~P(기타 AI 분석)
- V1 로드맵: 57개 항목 — MCP, Hooks, CAI 2.0, 코드 실행, 웹 검색
- V2 로드맵: 175개 항목 — Agent Teams, Agent SDK, Canvas, 감정 분석, Computer Use
- V3+ 로드맵: 84개 항목 — PARL, MoA, Dream Mode, 신경기호(Neurosymbolic) 추론

### 주요 항목 상세 (5개)

#### 1) S7-A-001: 멀티 에이전트 팀 리더/멤버 분리 (Multi-Agent Team Architecture)
- **목적**: AI 에이전트를 팀처럼 운영 — 리더가 작업을 분배하고 멤버가 실행
- **기술**: Redis Pub/Sub, MessageBus 패턴 (메시지 버스 = 에이전트 간 메시지 전달 통로)
- **버전**: V2 | 우선순위: CRITICAL
- **관련 모듈**: D2.0-05 AGENT_WORKFLOW, D2.0-02 ORANGE_CORE

#### 2) S7-A-014: MCP 클라이언트/서버 양방향 구현 (MCP Bidirectional Implementation)
- **목적**: MCP(Model Context Protocol = AI 도구 연결 표준 규격)를 통해 외부 도구를 자유롭게 연결
- **기술**: Streamable HTTP 전송, OAuth 2.1 인증
- **버전**: V1 | 우선순위: CRITICAL
- **관련 모듈**: D2.0-04, MCP 서버 아키텍처

#### 3) S7-A-020: Hooks 생명주기 시스템 (Hooks Lifecycle System)
- **목적**: AI 동작의 각 단계(도구 사용 전/후, 세션 시작 등)에 사용자 정의 로직을 삽입
- **기술**: Python asyncio 이벤트 루프, YAML/JSON 후크 정의
- **LOCK**: `.vamos/hooks/` 설정 형식은 `.toml`이 정식 (PHASE_B4 LOCK) — **변경 불가**
- **버전**: V1 | 우선순위: CRITICAL

#### 4) S7-A-024: Constitutional AI 2.0 (헌법적 AI = 이유 기반 정렬)
- **목적**: AI가 단순 규칙이 아닌 **이유를 이해**하고 행동하도록 정렬
- **4단계 우선순위**: 안전 > 윤리 > 규정 준수 > 유용성
- **버전**: V1 | 우선순위: CRITICAL
- **관련 모듈**: D2.0-07

#### 5) S7-A-009: 자체 Agent SDK (Self-built Agent SDK)
- **목적**: VamosAgent 기본 클래스로 누구나 에이전트를 만들 수 있는 개발 키트
- **기술**: Python SDK + Pydantic + asyncio, 내장 도구, 자동 컨텍스트 관리
- **버전**: V2 | 우선순위: CRITICAL

## §44.2 Cat-B: 대화/프로세스 (Conversation Process)

### 비유

> **전화 상담원의 응대 매뉴얼**과 같습니다. 고객(사용자)의 말투, 감정, 대화 흐름을 파악해서 가장 적절한 방식으로 응답하는 전체 프로세스를 설계합니다.

### 항목 수: S7B-001 ~ S7B-035 (35개)

### 핵심 내용 요약

10개 AI 서비스의 대화 프로세스를 88개 항목으로 비교 분석한 결과입니다. 이미 적용된 것 31개(35.2%), 설계만 된 것 22개(25.0%), **미적용(보강 대상) 35개**(39.8%)로 분류됩니다. VAMOS만의 고유 강점 16개(5-Gate, Decision Object, QoD, EVX 체인 등)를 포함합니다.

- 감정 분석 통합 아키텍처: 텍스트(V1) → 음성(V2) → 표정(V3)
- 우선순위 분포: CRITICAL 7, HIGH 17, MEDIUM 7, LOW 4

### 주요 항목 상세 (5개)

#### 1) S7B-001: 적응형 사고 (Adaptive Thinking)
- **목적**: 질문 난이도에 따라 AI의 **사고 깊이를 자동 조절** (쉬운 질문 = 빠른 답변, 어려운 질문 = 깊은 분석)
- **구조**: difficulty_classifier → think_depth_policy (auto/shallow/deep/extended)
- **버전**: V1=규칙 기반, V2=ML 분류기 | 우선순위: CRITICAL

#### 2) S7B-002: 텍스트 감정 감지 모듈 (Text Emotion Detection)
- **목적**: 사용자 메시지에서 감정 상태를 자동 파악
- **기술**: VADER/KoBERT(한국어 특화 AI 모델)
- **출력**: EmotionFrame — 감지된 감정, 신뢰도, 문화적 맥락 포함
- **버전**: V1 | 우선순위: CRITICAL

#### 3) S7B-003: 감정 기반 응답 톤 조절 (Adaptive Response Tone)
- **목적**: 감지된 감정에 맞춰 응답의 **톤/상세도/공감 수준**을 자동 조절
- **출력**: ResponseAdaptation 스키마 (tone, detail_level, empathy_level)
- **버전**: V1 | 우선순위: CRITICAL

#### 4) S7B-005: Hook 생명주기 시스템 (Hook Lifecycle)
- Cat-A의 S7-A-020과 동일한 시스템의 대화 프로세스 적용 버전
- **버전**: V1 | 우선순위: CRITICAL

#### 5) S7B-006: 실시간 웹 검색 통합 (Real-time Web Search / Search Grounding)
- **목적**: 대화 중 최신 정보가 필요하면 자동으로 웹 검색 수행
- **기술**: Tavily/Serper MCP 연동, 월 $5~20
- **버전**: V1 | 우선순위: CRITICAL

> **5층짜리 수납 시스템**에 비유됩니다. 1층(L0)은 지금 쓰고 있는 **작업 책상** 위, 2층(L1)은 이번 주 **서랍**, 3층(L2)은 **프로젝트 캐비닛**, 4층(L3)은 **영구 보관 도서관**, 5층(L4)은 **압축 보관 창고**입니다.

### 항목 수: S7D-001 ~ S7D-082 (82개)

### 핵심 내용 요약

9개 파트로 구성됩니다: 메모리 비교(8), 벡터DB(10), 그래프DB/KG(8), 임베딩 모델(8), 5-Layer 메모리 설계(12), 캐싱(8), RAG 인덱싱 파이프라인(10), 데이터 수명주기/프라이버시(10), 저장소 비용 최적화(8).

VAMOS의 **5-Layer 메모리**는 경쟁사를 크게 앞섭니다:
| AI 서비스 | 메모리 계층 수 |
|-----------|---------------|
| **VAMOS** | **5계층 (L0~L4)** |
| ChatGPT | 4계층 |
| Claude | 2계층 |
| Gemini | 1.5계층 |
| Perplexity | 0계층 |

### 5-Layer 메모리 아키텍처

| 계층 | 이름 | 비유 | 저장소 | 보존 기간 | 용량 |
|------|------|------|--------|-----------|------|
| L0 | Session Buffer (세션 버퍼) | 작업 책상 | 메모리(RAM) | 세션 중 | 200K 토큰 |
| L1 | Short-Term (단기 기억) | 이번 주 서랍 | SQLite+JSON | 7일 TTL | 1,000항목 |
| L2 | Project (프로젝트 기억) | 프로젝트 캐비닛 | SQLite+VectorDB | 90일 TTL | 10,000항목 |
| L3 | Long-Term (장기 기억) | 영구 도서관 | SQLite+VectorDB+KG | 영구 | 무제한 |
| L4 | Archive (압축 보관) | 압축 창고 | ZSTD 압축 SQLite | 영구 | 무제한 |

> ⚠️ **[PART1 ST-01]**: 정식(Canonical) 메모리 계층은 4계층(L0~L3)입니다. L4 Archive는 V2+ 참조용입니다.
