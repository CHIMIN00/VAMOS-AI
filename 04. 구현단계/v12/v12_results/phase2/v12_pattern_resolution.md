## 패턴별 상세

### Pattern 1: v10 연쇄 미갱신 (Pattern A)

**현재 상태**: RESOLVED

**Phase -1A 확인 내용**:
- §6.13 합계: ~506으로 갱신 (V0=~41, V1=~281, V2=~117, V3=~67)
- §7 GO/NO-GO: 총 64건(V0=16, V1=22, V2=14, V3=12)으로 갱신
- §7 Stage Gate: 총 204건(V0=58, V1=66, V2=43, V3=37)으로 갱신
- §6.3 테스트: ~128개 항목으로 갱신

**v12 Phase 1 매핑에서의 잔여 이슈 확인**:
- M-6(§1+§7 매핑)에서 GO/NO-GO 총 64건 확인, 그러나 READINESS_GUIDE의 62건과 2건 차이 발견 (§6.4 READINESS_GUIDE는 v22 PARL 추가분 2건 미반영 가능성)
- 이는 Pattern A의 재발이 아닌 READINESS_GUIDE 문서의 별도 갱신 이슈. PART2 본문 수치는 정확

**재발 가능성 검토**:
- v25.2.0에서 수치 갱신이 완료된 상태이므로, 향후 모듈 추가/삭제 없이 현행 유지 시 재발 위험 없음
- 만약 v26 이후 모듈 추가 시, 동일 패턴(수치 미연동) 재발 가능. §6.13 합계, §7 체크리스트, §6.3 테스트 수를 동시 갱신하는 체크리스트가 이미 §7에 존재

**잔여 조치**: 없음. READINESS_GUIDE 62 vs 64 차이는 Phase 3 수정 시 확인 권고 (LOW 우선순위)

### Pattern 3: V1 구조 고립

**현재 상태**: RESOLVED

**Phase -1A 확인 내용**:
- §3(V1) 6개 Phase 전부에 7요소(작업내용/산출물참조/실행가이드/AI프롬프트/규칙/참조SOT/완료검증) 추가 완료
- §2(V0)/§4(V2)/§5(V3)와 동일한 균일 구조 달성
- 각 Phase별 완료검증 항목 수: Phase 1(10), Phase 2(12), Phase 3(11), Phase 4(12), Phase 5(11), Phase 6(10)

**v12 Phase 1 매핑에서의 잔여 이슈 확인**:
- M-2(V1→§3 매핑): 1,582건 처리, PARTIAL 1,273건(80.5%)
- PARTIAL 비율이 높은 이유: V1 Feature 1,582건 중 상당수가 §3 Phase 테이블에는 1행 요약 + §6 참조 구조. 이는 V1 구조 고립(7요소 부재)과는 별개 이슈
- V1의 §6 참조 48건은 v10 확장 항목의 파일경로 칼럼에서 사용되며, 이는 §6 상세와의 정상적 연결

**재발 가능성 검토**:
- 7요소 구조가 §3 전 Phase에 정착되어 있으므로 구조적 재발 가능성 없음
- PARTIAL 1,273건은 정보 밀도 이슈이며, V1 구조 고립(하위구조 부재)과는 구별됨

**잔여 조치**: 없음. V1 PARTIAL 비율은 별도 이슈(정보 밀도)로 Phase 2 다른 작업에서 분석

### Pattern 4: V3 과적재 (§6 참조 비율)

**현재 상태**: RESOLVED (경과관찰 권고)

**Phase -1A 확인 내용**:
- §5(V3) 전체에서 "§6 참조" 0건 확인
- V3 Phase 테이블이 모듈명/시리즈/구현내용을 직접 기술하는 자기완결적 구조로 전환
- "§6 참조" 패턴은 §3(V1) 48건, §4(V2) 7건에만 존재

**v12 Phase 1 매핑에서의 잔여 이슈 확인**:
- M-4(V3→§5 매핑): 88건 처리, PARTIAL 19건(21.6%), MISSING 12건(13.6%)
- V3 PARTIAL 19건 중 9건(47.4%)이 §6 참조를 이유로 PARTIAL 판정 — 이는 §5 테이블이 아닌 M-4 매핑 과정에서 Feature 상세가 §6에만 존재하는 경우
- 자기완결성: v11 당시 2.5/5 → v25.2.0에서 **3.5/5**로 개선. 그러나 완전하지 않음
- Phase 1 보고서 §6.2: "Phase 3 v23 항목은 1줄 설명 → V1/V2 대비 상세도 저하" 지적

**재발 가능성 검토**:
- "§6 참조" 명시적 위임은 0건으로 해소되었으나, V3 Phase 테이블의 정보 밀도가 V1/V2 대비 낮은 상태 지속
- v23 확장 항목의 1줄 설명은 구현자 관점에서 정보 부족 가능성 있음
- 향후 V3 항목 추가 시, 1줄 설명 패턴이 반복될 경우 사실상 과적재의 변형(정보 희석) 재발 가능

**잔여 조치**:
- 자기완결성 3.5/5는 허용 범위이나, Phase 3 수정 시 V3-Phase 2(22그룹 밀집)/Phase 3의 v23 항목 상세도를 점검하는 것을 권고 (LOW 우선순위)
- PART2 수정이 아닌 모니터링 수준의 경과관찰 사항

**재발 가능성 검토**:

- 재발 가능성: 낮음 (새로운 카테고리가 추가되지 않는 한)

**잔여 조치**: 없음. M-3 MISSING 177건은 별도 이슈(§4 테이블 미기재)로 Phase 2 다른 작업에서 분석
