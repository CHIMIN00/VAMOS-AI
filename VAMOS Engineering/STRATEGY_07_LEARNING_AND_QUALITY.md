## 1. 전략 개요

```
핵심 원칙:
  "같은 실수를 두 번 하지 않고, 같은 조건에서 같은 결과가 나와야 한다"

2가지 기법:
  Retrospective: Phase마다 "잘된 것/안된 것/바꿀 것" 기록 (A11)
  Reproducibility: 검증 결과를 동일 조건에서 재현 가능하게 (A17)
```

## 2. A11: Phase 회고 + 포스트모템

### 2.1 문제 정의

```
현재: Phase 끝나면 바로 다음 Phase 시작
      "뭐가 잘 됐고 안 됐는지" 돌아보지 않음
      → 같은 실수를 다음 Phase에서 반복 위험
```

### 2.2 전략: 3-1 회고법

```
Phase 완료 시 (5분):
  잘된 것 3가지:   구체적으로 기록
  안된 것 3가지:   구체적으로 기록 + 원인
  다음에 바꿀 것 1가지: 구체적 조치

저장 위치: VAMOS Engineering/decisions/phaseN_retro.md
```

### 2.3 Phase별 회고 템플릿

```markdown
# Phase [N] 회고

> 완료일: 2026-XX-XX
> 소요: 약 X일 (실작업 X시간)

## 잘된 것 3가지
1. [구체적 성공 사항]
2. [구체적 성공 사항]
3. [구체적 성공 사항]

## 안된 것 3가지
1. [문제] — 원인: [왜 발생했는지]
2. [문제] — 원인: [왜 발생했는지]
3. [문제] — 원인: [왜 발생했는지]

## 다음에 바꿀 것 1가지
[구체적으로 Phase N+1에서 이렇게 하겠다]

## Phase 실패가 있었다면 (포스트모템)
- 실패 시점: Phase N-X 작업 중
- 증상: [무엇이 안 됐는지]
- 근본 원인: [왜 안 됐는지]
- 대응: [어떻게 해결했는지]
- 재발 방지: [같은 일이 안 일어나려면]
```

### 2.4 포스트모템 규칙 (SRE 방식)

```
원칙: Blameless (비난 금지)
  → "내가 잘못했다"가 아니라 "시스템/프로세스에 뭐가 부족했다"
  → 사람 탓이 아니라 구조 개선

기록 대상:
  Phase 실패 시 (A1 복구 발동 시)
  → 왜 실패했는지 + 어떻게 복구했는지 + 재발 방지책

기록 안 해도 되는 것:
  정상 완료된 Phase → 간단한 3-1 회고만
```

## 3. A17: 재현성 (Reproducibility)

### 3.1 문제 정의

```
현재: D1에서 /sot-conflict scan 결과 CONFLICT 3건
      한 달 후 같은 명령 → CONFLICT 5건?
      → SOT가 바뀌었나? 스킬이 바뀌었나? 둘 다?
      → 이전 결과를 재현할 수 없으면 비교 불가
```

### 3.2 재현성 2가지 유형

```
유형 1: 결정론적 검증 (DV-1~9, /sot-conflict 수치 비교 등)
  → 같은 입력 + 같은 스킬 = 항상 같은 결과
  → 재현성 자동 보장
  → 추가 조치 불필요

유형 2: AI 기반 검증 (SSV-1~3, /fact-audit, /hallucination-check 등)
  → AI는 매번 다른 답을 줄 수 있음
  → 재현성 보장 안 됨
  → 추가 조치 필요
```

### 3.3 전략: 유형 2 재현성 보장

```
Phase 1 (D1 AI 검증 시):
  ① seed 고정 가능한 경우 → temperature=0 + seed 고정
  ② seed 고정 불가한 경우 → 3회 실행 → 3회 중 2회 이상 동일 → PASS
  ③ 모든 검증 결과에 메타데이터 기록:
     - 실행 일시 (timestamp)
     - 사용 스킬 이름 + 버전
     - 입력 파일 해시 (어떤 SOT 상태에서 실행했는지)
     - 결과 (PASS/FAIL + 상세)

Phase 5 (멱등성 테스트):
  ① 동일 입력 3회 실행 → Decision 구조 동일 확인
  ② seed=42 고정 → 결과 비교
  ③ 결과 저장: eval_results.json에 seed + 반복 횟수 기록
```

### 3.4 검증 결과 저장 포맷

```json
{
  "verification_id": "D1-2026-04-XX-001",
  "timestamp": "2026-04-XX T14:30:00Z",
  "skill": "/sot-conflict",
  "skill_version": "v13",
  "command": "scan",
  "input_hash": "sha256:abc123...",
  "input_description": "SOT 68개 파일 (2026-04-XX 기준선)",
  "result": {
    "verdict": "CONFLICT",
    "count": 3,
    "details": ["..."]
  },
  "reproducibility": {
    "type": "deterministic",
    "repeat_count": 1,
    "note": "결정론적 검증 — 재실행 시 동일 결과 보장"
  }
}
```

### 3.5 재현성 확인 체크리스트

```
Phase 1 완료 시:
  □ D1 결정론적 검증(1-2~1-5) 결과에 input_hash 기록?
  □ D1 AI 검증(1-6, 1-7) 결과에 3회 반복 결과 기록?
  □ integrity_snapshot.json에 전체 SOT 해시 기록?

Phase 5 완료 시:
  □ 멱등성 테스트 seed 고정 3회 실행?
  □ eval_results.json에 seed + 반복 횟수 기록?
  □ baseline 대비 비교 가능한 포맷?
```
