# Phase 0 회고 (A11)

> **작성일**: 2026-04-04
> **Phase**: 0 (자산 인벤토리 + 전체 그림 확정)
> **세션**: P0-0 ~ P0-4 (총 5세션)
> **근거**: STRATEGY_07 A11 — Phase 완료 시 회고

---

## 잘된 것 3가지

### 1. 매트릭스 프레임워크의 조기 확립

STRATEGY_08 v1.1 매트릭스를 Phase 0에서 완성하여, 이후 모든 엔지니어링 활동의 **분류 기준**이 사전에 확정되었다. 20개 셀 × I/O 계약이 명확하여 Phase 1 이후 작업 범위가 즉시 식별 가능하다.

### 2. 미연동 8건의 체계적 해소

P0-3에서 매트릭스 v1.0→v1.1 갱신 시, 8건의 미연동 항목(Obsidian, ROADMAP, VERIFICATION SYSTEM, GO/NO-GO, PART1, LOCK REGISTRY, CLAUDE 보강전략, 5 STRATEGY 문서)을 전부 매핑하여 "빠진 자산 없음" 상태를 달성했다.

### 3. P0-2 산출물의 선제 생산

CLAUDE.md 구조 명세(§1~§28), Obsidian×매트릭스 매핑(17폴더×20셀), 설계 자산 전체 맵 — 세 문서를 Phase 0에서 미리 정의하여 Phase 2 진입 시 즉시 작업 가능한 상태를 확보했다.

---

## 안된 것 3가지

### 1. SOT 2 파일 수 변동 미예측

P0-0 시점 SOT 2 기준 648개 파일이었으나, P0-3 시점 710개로 증가하였다. R02(SOT↔SOT 2 불일치)/R11(검증 범위 확장) 리스크가 현실화될 가능성이 높아졌다. Phase 1 D1 실행 전 최신 파일 수 재확인 필요 (현행 2,654 — Phase4 확장 후 실측).

### 2. Could 항목(0-4, 0-5) 후순위 밀림

5개 계획서 목차와 하네스 역참조는 Could로 분류되어 P0-4까지 밀렸다. 시간 압박으로 목차만 초안 수준으로 작성 — Phase 해당 시점에 상세화가 반드시 필요.

### 3. decisions/ 폴더 초기 미생성

의사결정 기록(A6) 저장소인 decisions/ 폴더가 Phase 0 진행 중 생성되지 않아, P0-4 Gate에서 뒤늦게 생성했다. Phase 시작 시 저장소 구조를 먼저 확보하는 습관 필요.
