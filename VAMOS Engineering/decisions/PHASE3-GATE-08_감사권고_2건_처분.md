## 권고 1 — OpenAI 구키 revoke: **통보 미수신 유지 — 등재 조건 미충족 (트리거 유지)**

- 근거: `_targets/PHASE2_환경리포트.md` §5 "revoke 통보 미수신 — 기록만(현행 키 HTTP 200 유효)" + PHASE2-DEC-03 §2(2026-06-12 재실측 ✅) — 이후 신규 통보 기록 없음.
- 성격: 비차단(Opus INFO 권고). 사용자 액션 대기 항목 — AI가 대신 판단·등재할 수 없는 외부 사실.

## 권고 2 — main 병합 정책: **확정 (Phase 게이트 ff-only 동기화)**

**결정**: ① 작업은 phase 브랜치(현 `phase01-targeted-fixes`)에서 지속한다. ② **각 Phase 게이트의 완료 커밋·태그 이후 `main`을 `--ff-only`로 동기화하고 push한다.** ③ 소급 1회차 동기화(현 29커밋분)는 본 게이트(P3-0) 완료 커밋 직후 집행한다.

- 근거 (2026-06-12 실측): `main..phase01-targeted-fixes` = **29 / 역방향 0** — main(HEAD `1c3cede`, 2026-06-05)은 순수 후행이며 분기 0, fast-forward만으로 동기화 가능(병합 충돌·이력 개변 위험 없음). 원격 `origin/main` 존재.
- 이유: ①phase2-complete 등 게이트 태그가 push되어 있는데 main이 D1 시점에 정지해 있어, 저장소 기본 브랜치가 검증된 최신 상태를 반영하지 못함 ②ff-only 정책은 phase 브랜치의 검증 이력을 그대로 보존(squash/rebase 없음 — A2 태그 패턴과 정합).
- 기각 대안: (i) 명시적 보류 — 분기 0으로 위험이 없어 보류 사유 부재. (ii) 게이트마다 merge commit — ff 가능 상황에서 불필요한 이력 노드 생성.
