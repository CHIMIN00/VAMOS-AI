# PHASE3-GATE-05: STEP7 V1 라벨 ↔ PART2 V2/V3 배치 reconcile 방침 (1건)

> **결정일**: 2026-06-12 (P3-0 미결정 게이트 ⑤) · **포맷**: A6
> **배경**: 검증보고서 §4.3 잔여 구조 리스크 — STEP7 R1~R3이 V1로 라벨한 항목 일부(예: I-12 워크플로 빌더, RPA, EmotionAI 본대)가 PART2에서 V2/V3(COND/EXP)로 배치되어, V1 GO 시점에 "STEP7 V1 항목 미구현" 논란 소지.

## 결정 — 단일 방침

**STEP7 R1~R3의 V1 라벨 항목이 PART2 §3~§5에서 V2/V3로 배치된 경우, 구현 시점은 PART2 배치가 정본이다.**

1. **판정 기준**: V1 GO/NO-GO 분모 = PART2 V1 스코프(§3 V1-Phase 1~6 + §7.2 22항목 + Stage Gate V1 66항). STEP7 V1 라벨은 **기능 카탈로그 라벨**로 유지되며 구현 시점 판정에 사용하지 않는다.
2. **근거 계층**: 정본 우선순위 "RULE 1.3 > PLAN 3.0 > DESIGN LOCK > 본문 > **스키마/STEP7**" — STEP7은 최하위 계층이고, 2층 구조 선언(로드맵 L355·L505 노트)이 PART2를 V0·V1 실행 상세 정본으로 지정.
3. **수용 경로**: V2/V3 배치 항목은 Phase 7-0/8-0 진입 게이트가 해당 사이클 작업 목록으로 수용한다(로드맵 Phase 7/8 정식 편입 2026-06-11). 5-4 SHELL 87(v23 확장)도 동일 경로(PHASE3-GATE-07 §c).
4. **문서 처리**: STEP7 R1~R3 원문의 V1 라벨은 무수정 유지(SOT 무단 수정 금지). 라벨↔배치 차이의 개별 매핑 전수표는 P7-0에서 V2 작업 목록 작성 시 산출(현시점 ~700건 개별 대조는 불요 — 판정 기준이 확정되면 논란 자체가 차단됨).

## 검토 대안 (기각)
- STEP7 라벨 우선(미배치분 V1 편입): PART2 V1 게이트 분모 붕괴 + 정본 우선순위 역행 — 기각.
- STEP7 라벨 일괄 재작성: SOT 대규모 수정·이력 훼손 — 기각.

## 영향
- V1 GO/NO-GO(6-9) 판정 시 "STEP7 V1 라벨 미구현"은 FAIL 사유가 아님이 명문화됨. F-18(GATE-04)·5-4 SHELL 87(GATE-07 §c)과 일관된 단일 원칙.
