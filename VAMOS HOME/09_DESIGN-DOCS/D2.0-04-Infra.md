# D2.0-04 Infra

## 역할
INFRA-CORE 정본 — "OTHER BRAINS(모델/HW/실행자원)"를 교체 가능하게 추상화하고 비용/위험/품질 기준 런타임 라우팅 원칙을 고정 (버전 2.0.0, Active Canon, 최종 수정 2026-06-11).

## 핵심 섹션
- §2~3 Brain Engine Modularization + Brain Adapter Layer: CORE는 판단/제어, Adapter 뒤의 모델/HW는 CORE가 인식하지 않는 계약
- §4 HAL(Hardware Abstraction Layer) & 실행자원
- §5 Runtime Brain Routing: 비용/품질/위험 기반 선택 (I-5/I-8/I-6 연동)
- §6 멀티브레인/병렬 실행 정책: **병렬 태스크 최대 3** 등 운영 한계 반영
- §7 Tool 실행 계층 게이트 (크롤러/API/코드실행기/분석엔진)
- §0.1 개발 환경 셋업: V1 로컬(Python 3.11+, venv, SQLite, Chroma) / V2 Docker Compose / V3
