# D2.0-07 Safety-Cost

## 역할
VAMOS의 **안전/비용/승인/정책 체크(게이트)를 RULE 1.3 기반으로 설계 레벨로 고정**하는 정본 (버전 2.0.0, Active Canon, 최종 수정 2026-06-11). Approval/PolicyCheck/CostBudget/Downshift 스키마의 **SOT(D7)** — 어떤 외부 Agent 프레임워크 도입에도 D7이 우선.

## 핵심 섹션
- §1 Non-goal 절대 금지 7개 (RULE 1.3 §2 그대로 잠금: 실거래·해킹·의료법률 단정·민감정보 장기저장·저작권·P2 자동생성·위험 자동실행)
- §1.1/§10 Guardrails 3-Layer (ADD-013~015): NeMo Guardrails(입력)→Guardrails AI(처리)→LlamaGuard(출력) + Prompt Injection 방어(ADD-014) + ML 마스킹 V2
- §2 위험 등급 P0/P1/P2: P2 활성화=명시적 승인+세션별 재확인, HITL 타임아웃 **5분**(일반 승인 10분), 세션 종료 시 자동 OFF
- §4 비용 상한/다운시프트: V1/V2/V3 정본 표(LOCK) + 80% 경고 / 100% 차단(LOCK)
- §5 PolicyCheck 게이트 / §11 RLHF·Constitutional AI / §12 비용 대시보드 / §13 RBAC / §14 자율 운영 수준(MOD-027)
- §15 확장: ASL 계층, NLI Hallucination Detection(CRITICAL), RSP/Catastrophic Risk, Control Inversion 방지 등

## LOCK 하이라이트
- **외부 프레임워크 적용 원칙 (FREEZE)**: 에이전트 내부 임의 if문으로 정책 판정·승인·비용 차단 대체 금지(Gate 우회 위험) — Tool 호출/외부 실행/민감 처리/고비용 호출은 **PolicyCheck→CostBudget→Approval** 선행 필수
- §3.3 S-Module/E-Module 승인 규칙 표 (LOCK) / §4.1 비용 상한 정본 표 (LOCK) / §4.3 다운시프트 임계값 (LOCK)
- Non-goal 7개 + 비용 상한 + 승인 구조는 절대 불변 구역
