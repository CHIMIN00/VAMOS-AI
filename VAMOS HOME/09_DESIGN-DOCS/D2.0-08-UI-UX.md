# D2.0-08 UI-UX

## 역할
**Builder View + Hologram View의 UI 구조/패널/상태/이벤트 규칙 확정** 정본 (버전 2.0, Design Freeze). 설계 원칙 5: 2-View 분리 / 3단 패널 고정 / 근거 투명성(Evidence) / 모든 핵심 행위 LogEvent(event_type) / i18n 내장.

## 핵심 섹션
- §1 UI/UX 철학: 구조 가시성(Observability First), 이중 모드(Builder=만드는 창, Hologram=쓰는 창, 동일 run/trace/decision 공유), 근거 투명성(출처+인용 점프+QoD 상/중/하)
- §2~3 화면 구조 + 3단 패널: 좌(상태/정책/도메인) / 중(대화/작업/그래프) / 우(로그/근거/비용/승인/메모리)
- §4~5 UI 상태 머신 + 이벤트 명세: event_type 네임스페이스 `UI.*`, 사용자 행동→AI 반응→LogEvent 매핑
- §7~8 실패/폴백 UI 표현 + 접근 제어(승인 게이트/민감정보 마스킹 표시)
- §10 디자인 시스템 표준 / i18n: 기본 ko-KR, 보조 en-US, V2 ja-JP — `locales/{locale}/{ui|hologram|cli}.json`, LogEvent message는 영어 고정

## LOCK 하이라이트
- **변경 규칙**: UI 구조/정책/이벤트 스키마 변경은 승인(Approval) 기반으로만 허용 — 임의 수정 불가
- **Decision Lock의 UI 반영 (§1.4)**: ORANGE CORE 확정 결론/라우팅은 UI에서 수정 불가, 재시도는 근거/실행/포맷 축 한정(결론 변경 버튼 미제공)
- **DEC-008 확정**: D8 내 Provisional 빈 스키마 슬롯 전면 제거 → V3 확장은 D8-V3-SLOT-01~04(RESERVED) 체계로 명시 관리
- 승인 필요(P2/정책 변경/비용 임계)는 실행 HOLD 전환, 비용 80% 경고/초과 차단을 UI 행위 제한으로 구현
- i18n 하드코딩 금지 (MOD-022): 모든 UI 문구 i18n 키 관리
