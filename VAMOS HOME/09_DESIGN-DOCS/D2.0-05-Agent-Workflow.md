# D2.0-05 Agent Workflow

## 역할
VAMOS의 **Agent 실행 흐름(분해/우선순위/병렬/재시도/복구)과 로그/리포트 연결** 명세 (버전 2.0.0, Active Canon, 최종 수정 2026-06-11). I-5/I-12/I-13/I-6/I-9/I-18과 직결되는 Agent & AutoWorkflow 레이어의 정본.

## 핵심 섹션
- §1.1 Agent Mode 열거형(ADD-009): MANUAL / SEMI_AUTO / SUPERVISED_AUTO — V1 기본값 **SEMI_AUTO**, AUTONOMOUS는 V1 금지(IR-001)
- §2 Agent Pool: Task 분해 → BLUE NODE 위임, 병렬 최대 3(V3 옵션 10), 비용 상한이 최상위 제약
- §3~4 Workflow Engine + Fallback/Retry/Recovery: Soft loop/Hard loop, 위험·정책 위반 재시도 불가, 타임아웃 1회, Self-check <70 자동 1회 후 승인
- §5 Cooperative Agent Structure(Core→Node→Sub-agent) + ADD-024~027
- §7 워크플로우 표준: 5단계 Pipeline + 3단 출력 포맷 준수 의무
- §15~21 STEP7 통합: N-001~044(워크플로우/RPA), O-001~036(교육), I-001~106(AI 투자)

## LOCK 하이라이트
- **DEC-003 (LOCK)**: 승인 모드 = Allowlist 자동승인 확정 — 안전 도구 자동, P2 이상은 명시적 확인(IM-004)
- **P2 작업은 승인 없이 Queue 등록 불가** (RULE 1.3 §3.3 박제)
- Agent Mode별 Gate 연동표 고정: P2는 모든 모드에서 승인 필수, 비용 초과 시 다운시프트/중단
- 모드 변경은 세션 단위 + LogEvent(`agent.mode.changed`) 기록 의무
