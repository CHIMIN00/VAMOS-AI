# D2.0-06 Storage-Memory

## 역할
VAMOS의 **저장/메모리 원칙(4계층 L0~L3, 프로젝트 네임스페이스, 민감정보 저장 금지/확인)** 을 설계 레벨로 고정하는 정본 (버전 2.0.0, Active Canon, 최종 수정 2026-06-11). RULE 1.3 §7(4계층 메모리·project_id 필터·저장 최소화) 직계.

## 핵심 섹션
- §1.1 RAG 파이프라인 6단계 (LOCK): Collect→Chunk→Embed→Store→Retrieve→Generate, 운영 한계 문서 15개/청크 30개
- §2 메모리 계층 L0~L3 + B-Series 연결 (LOCK): L0 Session(B-4 Working) / L1 Project(B-1 Episodic) / L2 Long-term(B-3 Semantic) / L3 Procedural(B-2) — (Lx, By) 조합 매핑 규칙
- §3 저장 정책 Allow/Restrict/Deny + 마스킹 파이프라인: 로그 원문 저장 금지, input_summary/hint, ***/hash 마스킹
- §5 로그 저장소(I-9) 분리 원칙 / §6 보존·삭제·TTL·백업
- §7 스키마(MemoryRecord, SourceQoD 등 설계 레벨) / §8 레지스트리(PII_DETECTED, ST_ERR_VECTOR_DENIED 등)

## LOCK 하이라이트
- **(Lx, By) 매핑 규칙 LOCK** + L0~L3 정의는 PLAN 3.0 메모리 계층과 1:1 정합
- **L3(절차 메모리)는 저장/활성 모두 D7(07) 게이트 필수** — 행동에 직접 영향
- **Deny 판정 시 어떤 저장소에도 장기 저장 불가, 벡터 삽입 절대 금지**
- 루프/멀티툴 저장 폭주 방지 (FREEZE): 반복 루프 중 요약/메타/근거 링크만 저장, require_approval 상태에서 저장 HOLD
- 민감 데이터(PII/인증/의료/법률) 자동 저장 금지 — 항상 확인 또는 거부
